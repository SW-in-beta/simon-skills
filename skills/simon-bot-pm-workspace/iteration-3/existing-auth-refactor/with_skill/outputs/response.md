## Phase 0: Project Setup

### 0-A: 프로젝트 유형 판별

프로젝트 유형을 자동 판별했습니다:

> **Existing Project** (기존 프로젝트 리팩토링)
>
> 판별 근거: "모노레포", "현재 세션 기반인데 JWT로 바꾸고", "기존 API 호환성 유지", "관련 서비스가 5개" -- 기존 코드베이스에 대한 전환/리팩토링 작업입니다.

이 판별이 맞으시죠?

---

**Existing 프로젝트 사전 분석 제안:**

인증 시스템이 5개 서비스에 걸쳐 있고 호환성 유지가 필요하므로, 현재 코드베이스의 인증 구조를 먼저 체계적으로 파악하는 것이 중요합니다.

> **simon-bot-report로 사전 분석 보고서를 먼저 작성할까요?**
>
> 분석 보고서에서 다음을 파악합니다:
> - 현재 세션 기반 인증의 구현 구조 (미들웨어, 세션 저장소, 쿠키 관리 등)
> - 인증 관련 코드가 분포된 5개 서비스의 의존성 관계
> - 기존 API 엔드포인트 중 인증이 적용된 범위
> - 세션 관련 DB 스키마 / 저장소 구조
>
> 1. **Yes** -- 사전 분석 후 인터뷰 진행 (권장: 더 정확한 마이그레이션 계획 수립 가능)
> 2. **No** -- 바로 인터뷰 시작

### 0-B: 실행 모드 선택

각 기능 구현이 완료될 때마다 어떻게 진행할까요?

> 1. **자동 진행**: 완료 즉시 다음 작업으로 자동 진행 (중간 보고만)
> 2. **승인 후 진행**: 각 기능 완료 시 결과를 보고하고 승인 후 다음 작업 진행
>
> 인증 시스템은 보안에 민감하고 기존 서비스 호환성이 중요하므로, **승인 후 진행**을 권장드립니다. 각 단계에서 기존 API 호환성을 확인하시면서 진행할 수 있습니다.

## Phase 1: Spec-Driven Design -- 인터뷰 시작

### 1-A: Vision Interview (라운드 1/3~4)

말씀해주신 내용을 정리하면:

| 항목 | 파악된 내용 |
|------|-----------|
| **현재 상태** | 세션 기반 인증 (5개 서비스) |
| **목표** | JWT 기반 인증으로 전환 |
| **추가 기능** | OAuth2 지원 |
| **제약** | 기존 API 호환성 유지 |
| **규모** | 모노레포 내 5개 서비스 |

첫 메시지에서 핵심 방향은 명확하게 주셨습니다. 몇 가지 비즈니스 결정과 우선순위를 확인하겠습니다.

---

**질문 1: 마이그레이션 전략과 범위**

1. **마이그레이션 기간 동안 세션과 JWT를 동시에 지원하는 과도기가 필요한가요?** 예를 들어, 기존 클라이언트(웹/앱)가 세션 기반으로 동작하는 동안 새 클라이언트는 JWT를 사용하고, 일정 기간 후 세션을 완전히 제거하는 방식인지요? 아니면 한 번에 전환(빅뱅)할 계획인가요?

2. **5개 서비스가 구체적으로 어떤 서비스인지 알려주실 수 있나요?** (예: API 게이트웨이, 사용자 서비스, 결제 서비스 등) 서비스 간 인증 정보가 어떻게 공유되고 있는지도 궁금합니다.

3. **"기존 API 호환성 유지"의 범위는 어디까지인가요?**
   - (a) API 엔드포인트 URL과 응답 포맷은 유지하되 인증 방식만 변경
   - (b) 기존 세션 기반 API도 일정 기간 병행 운영 (deprecation period)
   - (c) 외부 파트너/서드파티가 사용하는 API가 있어서 반드시 하위 호환성 보장

---

**질문 2: OAuth2와 인증 요구사항**

1. **OAuth2는 Provider(우리가 OAuth2 서버가 되는 것)인가요, Consumer(Google/Kakao 등 소셜 로그인 연동)인가요, 아니면 둘 다인가요?**

2. **JWT 토큰 관련 세부 요구사항이 있나요?**
   - Access Token + Refresh Token 구조?
   - 토큰 만료 시간 정책? (예: access 15분, refresh 7일)
   - 토큰 블랙리스트/강제 만료 기능 필요 여부?

3. **현재 인증과 관련된 기존 문제점이나 개선하고 싶은 점이 있나요?** (예: 세션 저장소 확장성 문제, 마이크로서비스 간 인증 전파 어려움, 모바일 앱 지원 불편 등) -- 이 부분이 리팩토링의 핵심 동기를 이해하는 데 도움이 됩니다.

---

이 6개 질문에 대한 답변을 주시면, 나머지 세부사항(우선순위, 엣지 케이스)을 다음 라운드에서 빠르게 확인하고 Spec 작성으로 넘어가겠습니다.
