## Phase 1: Spec-Driven Design -- Vision Interview 시작

소셜 미디어 앱을 만드시려는 거군요! 피드, 팔로우, 좋아요, 댓글, DM까지 핵심 기능을 이미 명확하게 잡아주셨고, 사용자 규모도 초기 1만명에서 10만명까지 성장을 예상하신다는 것도 파악했습니다.

기술 스택은 이 단계에서는 다루지 않습니다 -- Phase 1-C에서 CTO/Dev Lead 전문가 패널이 요구사항을 분석한 뒤 최적의 스택을 추천해드릴 예정입니다. 지금은 **무엇을** 만들지에 집중하겠습니다.

몇 가지 여쭤볼게요:

### Round 1: 비전 & 핵심 방향

**1. 이 앱의 핵심 타겟 사용자는 누구인가요?**
- 특정 커뮤니티/관심사 기반인가요? (예: 게이머, 독서 모임, 직장인, 지역 커뮤니티 등)
- 아니면 범용 소셜 미디어인가요?
- 타겟 연령대나 특성이 있나요?

**2. 피드에 어떤 종류의 콘텐츠를 올릴 수 있나요?**
- 텍스트만? 텍스트 + 이미지? 동영상도 포함?
- 게시물 길이 제한이 있나요? (트위터처럼 짧은 글 vs 블로그처럼 긴 글)

**3. DM은 1:1 대화만 필요한가요, 그룹 채팅도 필요한가요?**
- 실시간 채팅(메시지가 즉시 표시)이 필요한가요, 아니면 메일함 형태(새로고침 시 확인)로 충분한가요?

---

*이 질문들에 답변해주시면, 다음 라운드에서 각 기능의 세부 시나리오와 우선순위를 구체화하겠습니다. 이미 답변하신 내용(기능 목록, 사용자 규모)은 다시 묻지 않겠습니다.*

*참고: 인터뷰는 총 3-5 라운드(8-12개 질문) 정도로 진행되며, "충분하다"고 하시면 언제든 다음 단계로 넘어갑니다.*

## 내부 메모 (PM 작업 노트)

### 이미 파악된 정보
- **프로젝트 유형**: Greenfield (새 프로젝트)
- **핵심 기능 (사용자 명시)**: 피드, 팔로우, 좋아요, 댓글, DM
- **사용자 규모**: 초기 1만명, 성장 목표 10만명
- **기술 스택 선호**: 없음 (모름 -- CTO 패널 추천 필요)
- **규모 분류**: Medium (1K-100K) -- 모듈러 모놀리스, PostgreSQL + Redis 패턴 후보

### 아직 파악 필요한 정보
- 타겟 사용자 / 앱의 차별점
- 콘텐츠 유형 (텍스트/이미지/동영상)
- 피드 알고리즘 (시간순 vs 추천)
- DM 실시간 여부 및 그룹 채팅 필요성
- 팔로우 모델 (일방향 vs 양방향/친구 요청)
- 알림 시스템 필요 여부
- 인증 방식 (이메일/소셜 로그인)
- 배포 환경 및 시간 제약
- 사용자의 프로그래밍 역량/선호 언어
- Must-have vs Nice-to-have 우선순위 확정

### 예상 인터뷰 로드맵
- **Round 1** (현재): 비전, 콘텐츠 유형, DM 형태
- **Round 2**: 피드 동작 상세, 팔로우 모델, 좋아요/댓글 세부사항
- **Round 3**: 알림, 인증, 프로필 기능, 검색/탐색
- **Round 4**: 우선순위 확정 (P1/P2/P3), 배포 환경, MVP 기한
- **Round 5** (필요시): 엣지 케이스 확인, 최종 정리

### Phase 1-C를 위한 사전 준비 (기술 스택 추천 시 입력 요소)
- 사용자가 기술 스택을 모르므로 CTO 에이전트가 Spec 요구사항 + 규모를 분석하여 최적의 스택을 추천해야 함
- 규모별 아키텍처 가이드에 따르면 Medium(1K-100K)이므로: 모듈러 모놀리스, PostgreSQL + Redis, 로드밸런서 패턴 검토
- DM 실시간 여부에 따라 WebSocket/SSE 필요성 결정 (기술 스택에 영향)
- 이미지/동영상 지원 여부에 따라 오브젝트 스토리지 필요성 결정
