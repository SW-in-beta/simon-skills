### 1-B: PRD 생성

사용자의 인터뷰 답변을 모두 수집한 후, `planner` 에이전트에게 인터뷰 결과를 전달하여 PRD를 작성합니다. 아래는 예시 PRD 골격입니다:

```markdown
# PRD: Go REST API Server (Community Platform)

## 1. Overview
- **비전**: Go + Gin + PostgreSQL 기반의 커뮤니티 REST API 서버
- **배경과 동기**: (인터뷰에서 파악한 동기)
- **대상 사용자**: (인터뷰에서 파악한 대상)

## 2. Goals & Non-Goals
- **Goals**: 사용자 관리, 게시판, 알림 기능이 포함된 RESTful API 서버 구축
- **Non-Goals**: 프론트엔드, 모바일 앱, 프로덕션 배포

## 3. User Stories
- 사용자로서, 회원가입하고 로그인하여 서비스를 이용할 수 있다
- 사용자로서, 게시글을 작성/수정/삭제할 수 있다
- 사용자로서, 게시글에 댓글을 달 수 있다
- 사용자로서, 내 글에 댓글이 달리면 알림을 받을 수 있다
- 관리자로서, 게시판을 관리할 수 있다

## 4. Feature Inventory
(Must-have / Nice-to-have 목록 -- 인터뷰 결과 기반)

## 5. Technical Requirements
- Language: Go 1.24+
- Framework: Gin
- Database: PostgreSQL
- ORM/SQL: (인터뷰 결과)
- Migration: (인터뷰 결과)
- Auth: JWT
- Testing: Go standard testing + (인터뷰 결과)
- Container: Docker + docker-compose

## 6. Constraints & Assumptions
- (인터뷰에서 파악한 제약사항)

## 7. Success Criteria
- 모든 Must-have 기능 구현 완료
- 전체 빌드 성공
- 테스트 통과
- API 문서화 완료
```

## 이후 진행 흐름 (Phase 2~6 요약)

사용자가 PRD를 승인하면, 이후 단계는 다음과 같이 자동으로 진행됩니다:

- **Phase 2**: `architect` 에이전트가 PRD를 분석하여 Feature 단위로 분해하고, 의존성 그래프와 실행 계획을 생성합니다.
- **Phase 3**: `executor` 에이전트가 프로젝트 뼈대(go.mod, 디렉토리 구조, 기본 설정)를 생성합니다.
- **Phase 4**: 각 Feature를 `simon-bot` 또는 `simon-bot-grind`에게 위임하여 TDD 기반으로 구현합니다.
- **Phase 5**: 통합 검증 (빌드, 테스트, 아키텍처 리뷰, 보안 리뷰)을 수행합니다.
- **Phase 6**: 최종 보고 및 사용자 가이드 리뷰 후 PR을 생성합니다.

## 시뮬레이션 종료 노트

위는 simon-bot-pm 스킬이 사용자의 요청에 대해 실제로 수행할 Phase 0과 Phase 1의 전체 시뮬레이션입니다.

**핵심 관찰 사항:**

1. **Phase 0**에서는 두 가지를 결정합니다:
   - 프로젝트 유형: 사용자의 요청이 "만들고 싶어"이므로 **Greenfield**로 자동 판별
   - 실행 모드: 사용자에게 "자동 진행" vs "승인 후 진행"을 질문

2. **Phase 1**에서는 6라운드에 걸친 인터뷰를 진행합니다:
   - 라운드 1: 비전, 목적, 대상 사용자, 프론트엔드 여부
   - 라운드 2: 사용자 관리 상세 (인증, 역할, 프로필)
   - 라운드 3: 게시판 상세 (유형, 기능, 권한)
   - 라운드 4: 알림 상세 (이벤트, 전달 방식, 읽음 관리)
   - 라운드 5: 기술 제약 (마이그레이션, ORM, 테스트, 배포)
   - 라운드 6: 우선순위 확정 (Must-have / Nice-to-have / Out of scope)

3. 인터뷰 원칙을 준수합니다:
   - 한 번에 2-3개 질문씩 제시
   - 사용자가 이미 답한 내용(Go, Gin, PostgreSQL)을 다시 묻지 않음
   - 비즈니스 결정과 우선순위에 집중
   - 기술적으로 추천 가능한 부분은 선택지를 제시

4. PRD 생성 후 반드시 사용자 리뷰/승인을 받은 뒤 다음 Phase로 진행합니다.
