### Interview Guard 적용 사항 (내부 메모)

이 인터뷰에서 다음 원칙을 적용했습니다:

1. **코드에서 알 수 있는 것은 묻지 않음**: simon-bot-report가 이미 분석한 기술 스택, 현재 세션 구현 방식, 디렉토리 구조, 기존 미들웨어 패턴 등은 질문하지 않고 보고서에서 참조
2. **비즈니스 결정에 집중**: 기술적 세부사항(어떤 라이브러리 쓸지 등)보다 "왜", "어디까지", "우선순위"에 집중
3. **한 번에 2-3개 질문**: 각 라운드에서 관련된 2-3개 질문을 묶어서 제시
4. **구체적 선택지 제공**: 열린 질문보다 선택지를 제시하여 사용자가 답하기 쉽게 함
5. **질문이 너무 많아지지 않도록**: 핵심 불확실성만 해소 -- 총 10개 질문으로 제한
