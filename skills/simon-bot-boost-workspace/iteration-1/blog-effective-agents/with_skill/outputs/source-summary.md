# Source Summary

- **제목**: Building Effective Agents
- **저자**: Anthropic (Engineering Blog)
- **URL**: https://www.anthropic.com/engineering/building-effective-agents
- **유형**: 블로그 (공식 기술 가이드)
- **핵심 주제**: LLM 에이전트 시스템의 설계 원칙, 워크플로 패턴, 구현 모범 사례

## 주요 아이디어

- **단순함 우선 (Simplicity First)**: 가장 단순한 솔루션부터 시작하고, 복잡성은 "demonstrably improves outcomes"일 때만 추가한다.
- **워크플로 vs 에이전트 구분**: 워크플로(사전 정의된 코드 경로로 LLM을 오케스트레이션)와 에이전트(LLM이 자체적으로 프로세스와 도구 사용을 동적으로 제어)를 명확히 구분한다.
- **프레임워크 경계 (Framework Caution)**: 프레임워크는 "extra layers of abstraction that can obscure the underlying prompts and responses"를 만든다. 프로덕션에서는 추상화를 줄이고 기본 컴포넌트로 빌드할 것을 권장한다.
- **투명성 (Transparency)**: 에이전트의 계획 단계를 명시적으로 보여주는 것이 중요하다.
- **ACI (Agent-Computer Interface)**: 도구 설계에 HCI 수준의 엄격함을 적용해야 한다. "poka-yoke"(실수 방지 설계) 원칙 적용.
- **환경으로부터의 Ground Truth**: 에이전트는 매 단계에서 "도구 호출 결과 또는 코드 실행" 같은 환경 피드백을 통해 진행 상황을 평가해야 한다.
- **Human-in-the-Loop**: 체크포인트에서 멈추거나 블로커 발생 시 인간 피드백을 구하는 패턴 권장. 최대 반복 횟수 같은 정지 조건 필요.
- **샌드박스 테스트**: 자율 에이전트는 비용 증가와 "compounding errors" 위험이 있으므로 샌드박스 환경에서 광범위한 테스트 필수.

## 워크플로 패턴 (5가지)

| 패턴 | 설명 | 적합한 상황 |
|------|------|------------|
| **Prompt Chaining** | 순차적 LLM 호출, 이전 출력을 다음 입력으로 | 고정된 하위 작업으로 깨끗하게 분해 가능한 작업 |
| **Routing** | 입력을 분류하여 전문화된 후속 작업으로 전달 | 별도 처리가 더 나은 뚜렷한 카테고리가 있는 복잡한 작업 |
| **Parallelization** | LLM이 동시에 작업, 결과를 프로그래밍으로 집계 | Sectioning(독립 하위 작업) 또는 Voting(다양한 출력 확보) |
| **Orchestrator-Workers** | 중앙 LLM이 동적으로 작업을 분해하고 워커에게 위임 | 하위 작업을 사전 예측할 수 없는 복잡한 작업 |
| **Evaluator-Optimizer** | 하나가 생성, 다른 하나가 평가/피드백을 루프로 제공 | 명확한 평가 기준이 있고 반복 개선이 측정 가능한 가치를 제공할 때 |

## 에이전트 핵심 특성

- LLM이 복잡한 입력 이해, 추론/계획, 도구 신뢰성 있는 사용, 에러 복구에 성숙해지면서 프로덕션에 등장
- 사용자 명령 또는 대화로 시작 → 독립적으로 계획하고 운영
- 필요한 단계 수를 예측하기 어렵고 고정 경로를 하드코딩할 수 없는 개방형 문제에 적합
- 비용 증가와 에러 축적 위험이 높으므로 가드레일 필수

## 도구 엔지니어링 원칙

- 도구에 "전체 프롬프트만큼의 엔지니어링 관심"을 기울여야 함
- 모델이 "thinking 토큰을 충분히 가지도록" 포맷 설계
- 인터넷에서 자연스럽게 보이는 텍스트에 가까운 포맷 유지
- 포맷 오버헤드(줄 번호 세기, 문자열 이스케이핑) 제거
- 예제 사용법, 엣지 케이스, 입력 요구사항, 다른 도구와의 경계 명시
- poka-yoke: 실수하기 어렵게 인자 설계 (예: 절대 경로 강제)

## 실전 도메인

- **고객 지원**: 대화 흐름 + 외부 정보/액션 접근이 자연스럽게 결합. 성공 기준이 명확.
- **코딩 에이전트**: 자동화 테스트로 솔루션 검증 가능. 테스트 결과를 피드백으로 반복 개선. 단, 인간 리뷰는 필수.
