# CheeR93 데모 시나리오 플랜

> 대상: 투자자 시연
> 예상 소요 시간: 15~20분
> 핵심 메시지: "마라톤 응원, 공유 링크 하나로 끝."

---

## 사전 준비 체크리스트

### 환경 설정
- [ ] `.env.local` 환경변수 설정 완료 (Supabase, Kakao Maps, VAPID, CoolSMS)
- [ ] `pnpm dev` 로컬 개발 서버 구동
- [ ] Supabase 프로젝트 활성화 상태 확인
- [ ] 시뮬레이션 데이터 사전 생성:
  ```bash
  npx tsx scripts/simulation/generate.ts --gpx supabase/seed-course.gpx
  ```
- [ ] 시뮬레이션 실행 준비 (별도 터미널):
  ```bash
  npx tsx scripts/simulation/runner.ts --speed 60
  ```

### 디바이스 준비
- [ ] 데모용 모바일 기기 1대 (응원자 뷰 -- 가장 인상적인 화면)
- [ ] 노트북 화면 (관리자 뷰 + 시뮬레이션 제어)
- [ ] 모바일 화면을 대형 스크린/모니터에 미러링 (AirPlay 등)

### 데이터 준비
- [ ] 시뮬레이션용 GPX 파일 준비 (supabase/seed-course.gpx 활용)
- [ ] 대회 live 상태 확인
- [ ] 주자 5명(배번 101~105) 등록 확인 -- generate.ts가 자동 생성

### ACT 2: 관리자 흐름 -- 대회 설정 (3분)

**대사:**
> "대회 관리자가 하는 일은 딱 세 가지입니다. 대회 만들고, 코스 올리고, 시작 버튼 누르기."

**[화면: 노트북 -- 관리자 뷰]**

#### 2-1. 카카오 로그인
- `/login` 접속
- 카카오 OAuth 로그인 버튼 클릭 -> 인증 -> `/admin`으로 리다이렉트

#### 2-2. 대회 목록 확인
- `/admin` -- 이미 시뮬레이션으로 생성된 "시뮬레이션 대회"가 목록에 표시
- **포인트**: "관리자가 자기 대회만 보입니다. RLS로 데이터 격리되어 있습니다."

#### 2-3. 대회 상세 페이지
- 대회 카드 클릭 -> `/admin/races/[raceId]`
- **보여줄 것**:
  - 대회 정보 카드 (대회명, 날짜, SmartChip ID, 총 거리)
  - 대회 상태 뱃지 ("진행 중" 빨간 뱃지)
  - GPX 코스 미리보기 (CoursePreview -- 지도에 코스 선 렌더링)
  - 주자 참가 링크: `/race/{smartchipId}/join`

**포인트:**
> "GPX 파일 하나만 올리면 코스 전체가 자동으로 파싱됩니다. 누적 거리 계산, km 지점 보간, 총 거리 산출까지 전부 자동이에요."

### ACT 3: 응원자 경험 -- 하이라이트 (8분) ★가장 중요★

**대사:**
> "이제 응원자 입장이 됩니다. 카카오톡으로 링크 하나 받았다고 생각해 주세요."

**[화면: 모바일 기기 (대형 스크린 미러링)]**

#### 3-1. 공유 링크 접속 (로그인 불필요)
- 모바일에서 `/race/{smartchipId}` 접속
- **보여줄 것**:
  - 로그인 화면 없이 바로 지도 진입
  - 상단 헤더: 대회명 (반투명 블러 배경)
  - Kakao Maps에 코스 Polyline 렌더링
  - 5km 간격 KmMarker 표시
  - 지도 로딩 중 "지도 로딩 중..." 스켈레톤 표시 후 전환

**포인트:**
> "회원가입 0초. 앱 설치 0초. 링크 클릭한 순간 이미 응원 준비 완료입니다."

#### 3-2. 시뮬레이션 시작 -- 실시간 주자 이동 ★핵심 데모★
- **별도 터미널에서** 시뮬레이션 실행:
  ```bash
  npx tsx scripts/simulation/runner.ts --speed 60
  ```
  (60배속 -- 30초마다 실시간 30분이 흐르는 효과)

- **보여줄 것**:
  - 주자 5명의 마커가 코스 위에 나타남
  - 마커에 배번, 이름, 페이스 표시 (예: "101 | 주자 101 | 4'00"/km")
  - **마커가 부드럽게 이동** (useMarkerInterpolation -- lerp 보간)
  - 각 주자가 서로 다른 속도로 달림 (4분~6분/km)
  - 연결 상태 표시 (ConnectionStatus -- "connected" 상태)

**포인트:**
> "마커가 뚝뚝 끊기지 않고 부드럽게 움직이는 거 보이시죠? 서버는 30초마다 위치를 보내주지만, 클라이언트에서 requestAnimationFrame으로 보간해서 60fps 애니메이션을 만듭니다."

#### 3-3. BottomSheet -- 주자 목록
- 하단 BottomSheet를 위로 스와이프
- **보여줄 것**:
  - 대회 상태 요약 (진행 중 / 총 주자 수 / 완주 수)
  - 주자 리스트 -- 배번, 이름, 현재 km, 현재 페이스
  - **실시간 갱신** -- km과 페이스가 계속 변화
  - 주자 탭 -> 해당 위치로 지도 이동 + 줌인

**포인트:**
> "한 손으로 스와이프해서 목록 보고, 탭하면 그 주자한테 바로 날아갑니다. 야외에서 한 손으로 조작하는 걸 기본 전제로 설계했습니다."

#### 3-4. 새로고침 버튼
- 우측 하단 플로팅 새로고침 버튼 클릭
- **포인트**: "5초 이내 재요청은 자동으로 차단됩니다. 서버 과부하 방지를 위한 쓰로틀링."

#### 3-5. 주자 접근 알림 (가능하면 시연)
- 시뮬레이션이 진행되면서 주자가 특정 km 지점을 통과할 때:
  - 인앱 AlertBanner가 상단에 슬라이드 인
  - "주자 101 -- 약 2분 후 도착 예정"
  - 5초 후 자동으로 MiniBadge로 최소화
  - 배너 탭 -> 해당 주자 위치로 지도 이동

**포인트:**
> "기존 서비스들은 '몇 km 통과했습니다'만 알려줍니다. CheeR93은 '지금 내가 서 있는 곳에 3분 후에 도착합니다'를 알려줍니다. 응원자 관점에서 정말로 쓸모있는 정보죠."

### ACT 5: 기술적 깊이 -- 투자자 질의응답 대비 (3분)

#### 5-1. 어댑터 패턴 설명

#### 5-2. SmartChip 위치 보간
> "SmartChip은 5km마다 매트를 통과할 때만 데이터가 나옵니다. 그 사이 구간은 '직전 구간 페이스로 달리고 있을 것이다'라고 가정하고 GPX 코스 위에서 보간합니다. 오차가 있을 수밖에 없으니까, 3단계 신뢰도를 UI로 명확히 보여줍니다."
- high: 실선 마커 (5분 이내)
- medium: 반투명 마커 (5~15분) -- "SmartChip-only 주자의 알림에는 '추정' 뱃지가 붙습니다"
- low: 마커 숨김, 텍스트 표시 (15분 초과)

#### 5-3. 비용 구조
> "개발 기간 비용은 0원입니다. 대회 당일에만 Supabase를 $25 플랜으로 올리고, SMS 비용 만원 정도. 대회 끝나면 다시 무료로 내립니다. 소규모 지역 마라톤 대회에 완벽한 구조입니다."

#### 5-4. 확장 가능성
> "현재는 소규모(주자 100명, 응원자 200명)이지만, 아키텍처 자체는 스케일아웃 가능합니다. Supabase Pro에서 Enterprise로, Fly.io Worker를 수평 확장하면 됩니다. 핵심은 Realtime Broadcast 전용 설계 -- Postgres Changes를 쓰지 않아서 DB 부하가 위치 업데이트에 비례하지 않습니다."

## 예상 질문 & 답변 가이드

### Q: "경쟁 서비스는 뭐가 있나요?"
> SmartChip 자체 웹사이트(수동 새로고침, 통과 기록만), Garmin Connect(러너 본인만 볼 수 있음), 외국 서비스 RaceJoy/RunGo(한국 미지원). CheeR93은 **응원자 중심** 설계로 차별화됩니다.

### Q: "Garmin 비공식 API가 막히면?"
> 어댑터 패턴으로 격리되어 있어 SmartChip으로 자동 폴백됩니다. Garmin 주자도 대회 배번이 있으므로 SmartChip 매트 통과 데이터로 전환 가능합니다.

### Q: "동시접속 한계는?"
> Supabase Free 200명, Pro 500명. 소규모 대회(100~200명)에 최적화. 그 이상은 Supabase Enterprise($599/월)로 확장 가능합니다.

### Q: "수익 모델은?"
> 현재는 비영리 전제. 확장 시: (1) 대회 운영 SaaS 월정액, (2) 응원 메시지 유료 기능, (3) 대회 기록 분석 리포트 구독.

### Q: "기존 대회 운영 시스템과 연동되나?"
> SmartChip과 Garmin이 한국 마라톤의 양대 데이터 소스. 이 두 가지를 모두 통합했으므로 대부분의 국내 대회에서 즉시 사용 가능합니다.

## 데모 실패 대비 백업 플랜

| 장애 상황 | 대응 |
|-----------|------|
| Supabase 연결 실패 | 로컬 Supabase (`supabase start`) 사용. config.toml + 마이그레이션 전부 준비되어 있음 |
| Kakao Maps API 장애 | TextRunnerStatus 컴포넌트로 폴백 가능 -- 텍스트 기반 주자 현황 테이블 보여주며 "이게 Graceful Degradation입니다"라고 설명 |
| 시뮬레이션 실행 실패 | 사전에 data.json을 생성해둠. runner.ts만 재실행하면 됨 |
| 모바일 미러링 실패 | 노트북에서 Chrome DevTools 모바일 뷰로 대체 |
| 실시간 Broadcast 안 됨 | 새로고침 버튼 (수동 REST fetch)으로 대체. "오프라인에서도 최신 데이터를 가져올 수 있습니다" |

## 인상적인 포인트 강조 순서 (투자자 감정선)

1. **공감** -- "마라톤 응원 가본 적 있죠?" (문제 공감)
2. **놀라움** -- 링크 클릭 하나로 지도 진입 (제로 프릭션)
3. **와우** -- 주자 5명이 동시에 코스 위를 부드럽게 달리는 실시간 지도
4. **실용성** -- "3분 후 도착" 알림 (행동 가능한 정보)
5. **신뢰** -- 아키텍처 깊이 (어댑터 패턴, 3단계 신뢰도, RLS 보안)
6. **경제성** -- 대회 당일 $25, 평소 $0
