# CheeR93 시연 계획서

**대상**: 투자자
**톤**: 영업/홍보 — 가치 중심, 인상적인 첫인상, 기술보다 사용자 경험
**총 시나리오**: 5개
**예상 소요**: 약 12~15분

---

## 시나리오 순서 전략

투자자 시연이므로 **임팩트 퍼스트** + **클라이맥스 배치**를 따른다:

1. **오프닝** — 응원자 관점에서 완성된 제품을 먼저 보여준다 (가장 인상적인 화면)
2. **관리자 흐름** — 대회 생성부터 시작까지, 운영 편의성 강조
3. **실시간 추적** — 주자 마커가 지도 위에서 실시간으로 이동하는 장면 (기술 하이라이트)
4. **알림 시스템** — ETA 기반 접근 알림 (핵심 가치 제안)
5. **관리자 대시보드** — 운영 모니터링 (클로징)

## 시나리오 1: 응원자 체험 — "로그인 없이 3초 만에 접속"

**관련 유저스토리**: US-5.1, US-5.2
**시연 목적**: 핵심 사용자(응원자)의 진입 장벽이 제로라는 것을 보여준다. 공유 링크 하나로 마라톤 현장이 손 안에.
**사전 조건**: 시뮬레이션 데이터 생성 완료 (대회 상태: live로 변경 전이어도 됨)
**예상 소요**: ~2분

### 동작 순서
1. 브라우저에서 `/race/{smartchipId}` 로 이동 (모바일 뷰포트 375px 권장)
2. 카카오맵에 코스 폴리라인(녹색 선)이 표시되는 것 확인
3. 5km 간격 마커가 코스 위에 표시되는 것 확인
4. 하단 BottomSheet를 위로 스와이프하여 펼침
5. 주자 목록(배번 원형 뱃지 + 이름)이 표시되는 것 확인
6. 대회 상태 배지("예정" 또는 "진행 중") 확인
7. BottomSheet에서 주자를 탭하면 지도가 해당 위치로 이동하는 것 시연

### 나레이션 포인트
- "응원자는 카카오톡으로 받은 링크를 탭하기만 하면 됩니다. 회원가입도 로그인도 필요 없습니다."
- "마라톤 대회 당일, 응원자의 95%는 야외에서 모바일로 접속합니다. 그래서 모바일 퍼스트로 설계했습니다."
- "하단 시트를 올리면 모든 주자 현황을 한눈에 볼 수 있고, 탭하면 해당 주자 위치로 바로 이동합니다."
- "카카오맵 위에 실제 마라톤 코스가 녹색 선으로 표시되어, 주자가 어디를 달리고 있는지 직관적으로 파악됩니다."

## 시나리오 2: 관리자 흐름 — "5분 만에 대회 준비 완료"

**관련 유저스토리**: US-1.1, US-2.1, US-2.3, US-2.4
**시연 목적**: 대회 운영자가 얼마나 쉽고 빠르게 대회를 세팅할 수 있는지 보여준다.
**사전 조건**: 카카오 계정으로 로그인 가능한 상태 (또는 이미 로그인)
**예상 소요**: ~3분

### 동작 순서
1. `/login` 페이지로 이동
2. "카카오 로그인" 버튼 클릭 → 카카오 OAuth 인증
3. `/admin` 대회 목록 페이지 도달 확인
4. "새 대회 만들기" 클릭 → `/admin/races/new`
5. 대회명 입력: "2026 잠실 하프마라톤"
6. 대회 날짜: 오늘 날짜 선택
7. SmartChip 대회 ID: "93001" 입력
8. "대회 생성" 버튼 클릭
9. → 대회 상세 페이지(`/admin/races/[raceId]`)로 자동 이동
10. 대회 정보 카드 확인 (대회명, 날짜, SmartChip ID, 주자 참가 링크)
11. GPX 업로드 영역에 `supabase/seed-course.gpx` 파일 드래그앤드롭
12. → 업로드 완료 후 지도에 코스 선 + km 마커 표시
13. "대회 시작 (Live)" 버튼 클릭 → 확인 다이얼로그에서 "확인"
14. → 상태 배지가 "예정"에서 "진행 중"(빨간색)으로 변경

### 나레이션 포인트
- "카카오 로그인 한 번이면 관리자가 됩니다. 별도 관리자 계정이 필요 없어요."
- "대회 생성은 세 가지 정보만 입력하면 됩니다 — 이름, 날짜, SmartChip ID."
- "GPX 파일은 Garmin, Strava 등에서 내보내기 한 코스 파일입니다. 드래그앤드롭으로 업로드하면 자동으로 코스가 파싱되고 지도에 미리보기가 표시됩니다."
- "대회 시작 버튼을 누르면, 이 순간부터 주자들의 실시간 위치 추적이 시작됩니다."
- "SmartChip ID가 곧 공유 코드입니다 — 이걸 카카오톡으로 보내면 누구나 `/race/93001`로 접속할 수 있어요."

## 시나리오 3: 실시간 추적 — "주자가 살아 움직입니다"

**관련 유저스토리**: US-4.1, US-4.3, US-4.4, US-5.2, US-8.2
**시연 목적**: 핵심 기술 — 실시간 위치 추적과 부드러운 마커 이동을 보여준다. 이것이 단순 텍스트 기록이 아닌 "라이브" 플랫폼임을 증명.
**사전 조건**: 시뮬레이션 데이터 생성 완료, 대회 live 상태
**예상 소요**: ~4분

### 동작 순서
1. 터미널에서 시뮬레이션 실행 (투자자에게 터미널도 보여주면 효과적):
   ```
   npx tsx scripts/simulation/runner.ts --speed 10
   ```
2. 브라우저에서 `/race/{smartchipId}` 페이지 이미 열어놓은 상태
3. → 주자 마커들이 지도 위에 나타나기 시작
4. 30초(시뮬 10배속이면 3초)마다 주자 마커가 부드럽게 이동하는 것 관찰
5. BottomSheet를 올려서 각 주자의 현재 km과 페이스가 실시간 갱신되는 것 확인
6. 빠른 주자(101번, 4'00"/km)와 느린 주자(105번, 6'00"/km)의 위치 차이 관찰
7. 주자 하나를 탭 → 지도 줌인 → 마커 이동을 가까이서 관찰
8. 시뮬레이션 터미널에서 split 통과 로그 확인 ("[101] 1km 통과 - 구간: 00:04:00, 누적: 00:04:00")
9. 새로고침 버튼(우측 하단) 클릭 → 즉시 최신 데이터 반영

### 나레이션 포인트
- "지금 보시는 것은 실제 시뮬레이션입니다. 5명의 가상 주자가 서로 다른 페이스로 달리고 있어요."
- "주목할 점은 마커가 순간이동하지 않고 **부드럽게 이동**한다는 것입니다. requestAnimationFrame과 선형 보간(lerp)으로 자연스러운 움직임을 구현했어요."
- "이 데이터는 Supabase Realtime을 통해 WebSocket으로 전달됩니다. 지연 시간은 1초 이내입니다."
- "현실에서는 Garmin GPS 직접 수집과 SmartChip 매트 통과 데이터 두 가지 소스를 통합합니다. 어댑터 패턴으로 데이터 소스 추가가 간편해요."
- "1km을 통과할 때마다 구간 기록이 자동으로 생성됩니다. 터미널 로그에서 확인하실 수 있어요."
- "동시 접속 100~200명 규모의 소규모 로컬 마라톤을 타겟으로 합니다. Supabase Free 플랜만으로도 대회 운영이 가능하고, 대회 당일만 Pro($25/월)로 전환하면 충분합니다."

## 시나리오 4: 응원 알림 — "주자가 다가옵니다!"

**관련 유저스토리**: US-5.3, US-5.4, US-7.3
**시연 목적**: 이 플랫폼의 핵심 가치 제안 — "응원 위치에서 기다리다 주자를 놓치는 일이 없다." ETA 기반 알림의 정확성과 UX.
**사전 조건**: 시뮬레이션 실행 중, 주자들이 아직 응원 위치(예: 3km)에 도달하지 않은 상태
**예상 소요**: ~3분

### 동작 순서
1. `/race/{smartchipId}` 페이지에서 (응원자 관점)
2. 응원 위치 설정 UI 접근 (CheerLocationForm)
   - "km 수동입력" 탭 선택
   - km 입력: `3` (3km 지점)
   - 닉네임 입력: "투자자 응원단"
   - "응원 위치 설정" 클릭
3. 지도에 내 응원 위치 마커 표시 확인
4. 시뮬레이션 진행 대기 — 주자 101번(가장 빠름, 4'00"/km)이 3km에 접근
5. → ETA ≤ 3분 되는 순간, 화면 상단에 알림 배너 슬라이드-인:
   - "주자 101 — 약 2분 후 도착 예정"
   - SmartChip 소스면 "추정" 노란 뱃지 표시
6. 알림 배너 탭 → 지도가 해당 주자 위치로 이동
7. 5초 후 자동 최소화 → 우측 상단 MiniBadge
8. 다른 주자들도 순차적으로 접근 → 알림 배너가 여러 개 쌓임 (최대 3개 + overflow 카운트)

### 나레이션 포인트
- "이것이 CheeR93의 핵심입니다. 응원자가 코스 위 특정 지점에서 기다리고 있으면, 주자가 3분 이내로 다가올 때 자동으로 알림을 보내줍니다."
- "더 이상 '언제 올까' 하고 초조하게 기다릴 필요가 없어요. 정확한 ETA와 함께 알려줍니다."
- "SmartChip 데이터의 경우 보간 추정이기 때문에 '추정' 표시가 붙습니다. 사용자에게 데이터 신뢰도를 투명하게 전달하는 것이 저희 원칙입니다."
- "알림은 인앱 배너뿐 아니라 Web Push(브라우저 백그라운드 알림)와 SMS로도 전달됩니다. 3채널 알림 시스템이에요."
- "같은 주자에 대해 30초 이내 중복 알림은 자동으로 차단됩니다."

## 시나리오 5: 관리자 대시보드 + 대회 종료 — "운영 관점의 한눈 모니터링"

**관련 유저스토리**: US-6.2, US-2.4 (live → finished)
**시연 목적**: 대회 운영자가 전체 상황을 실시간으로 파악할 수 있음을 보여준다. 클로징 — 완결된 제품임을 각인.
**사전 조건**: 시뮬레이션 진행 중 (일부 주자 완주 직전 또는 완주 후)
**예상 소요**: ~2분

### 동작 순서
1. `/admin/races/[raceId]/dashboard` 로 이동
2. 6개 StatCard 확인:
   - 활주 중: N명 (Activity 아이콘)
   - 완주: N명 (Award 아이콘)
   - DNF: 0명 (AlertTriangle 아이콘)
   - 평균 페이스: 5'XX"/km (Timer 아이콘)
   - Garmin: 2명 (Watch 아이콘)
   - SmartChip: 3명 (Cpu 아이콘)
3. 시뮬레이션 진행에 따라 완주 카운트가 실시간 변동하는 것 관찰
4. 시뮬레이션 완료 대기 (또는 이미 완료된 상태)
5. `/admin/races/[raceId]` 로 이동 → "대회 종료" 클릭 → 확인
6. → 상태 배지가 "진행 중"에서 "종료"로 변경
7. 응원자 화면으로 돌아가면 상태가 "종료"로 표시

### 나레이션 포인트

## 더미 데이터 계획 상세

### 정적 데이터 (시뮬레이션 generate.ts가 생성)
- races: 1개 대회
- course_points: GPX 파싱 결과 (~100개 이상 보간 지점)
- profiles + auth.users: 5명 주자 계정
- runners: 5명 (페이스 4'00"~6'00"/km, Garmin 2명 + SmartChip 3명)
- spectators: 3명 (1km, 3km, 5km 지점)

### 동적 데이터 (시뮬레이션 runner.ts가 실행 중 생성)
- tracking_positions: 30초 간격 INSERT (주자 5명 x 코스/30초 = ~수백 개)
- runner_current_positions: 매 스텝 UPSERT
- split_records: km 통과 시 INSERT
- Supabase Realtime: position_update, split_update Broadcast

### 스트리밍 데이터 패턴
시뮬레이션 runner.ts가 30초 실제 간격을 `--speed` 옵션으로 가속:
- `--speed 1`: 실시간 (30초 대기) — 실제 대회 느낌, 하지만 시연에는 너무 느림
- `--speed 10`: 10배속 (3초 대기) — **시연 권장** — 적절한 속도감
- `--speed 60`: 60배속 (0.5초 대기) — 빠른 데모용

### 현실감 있는 데이터
- 한국식 배번 (101~105)
- 주자 이름: "주자 101", "주자 102" 등 (시뮬레이션 기본값)
  - 더 인상적으로 하려면 시뮬레이션 코드에서 `display_name`을 "김민준", "이서윤" 등 실제 이름으로 변경 가능
- 페이스: 현실적 범위 (4'00"~6'00"/km, 시민 마라톤 수준)
- 위치: 실제 GPX 코스 좌표 위를 정확히 따라 이동

## 잠재 리스크 & 대비

| 리스크 | 대비 |
|--------|------|
| 카카오 OAuth 실패 | 시뮬레이션 스크립트가 데이터를 직접 생성하므로, 관리자 시연은 스킵하고 응원자 시연(로그인 불필요)에 집중 가능 |
| Supabase 접속 불가 | 사전에 Supabase 대시보드에서 상태 확인. 불가 시 기 녹화 영상 폴백 |
| Kakao Maps 로딩 실패 | API 키 확인. 실패 시 "지도를 불러올 수 없습니다" 메시지 표시됨 (Graceful Degradation) |
| 시뮬레이션 스크립트 오류 | 사전에 한 번 실행하여 data.json 생성 확인. generate → runner 순서 엄수 |
| Realtime 연결 끊김 | ConnectionStatus 배너가 자동 표시. 새로고침 버튼으로 REST 폴백 가능 |
| 주자 마커가 안 보임 | 주자 등록 시 `location_consent: true` 확인. 시뮬레이션은 기본 true |
