# CheeR93 데모 계획: 실시간 위치 추적 & 응원

총 4개 시나리오, 예상 소요 ~12분.

---

## 시나리오 구성 전략

사용자가 요청한 **실시간 위치 추적**과 **응원 기능**을 중심으로 구성한다.

**순서 전략**: 임팩트 퍼스트 + 클라이맥스 후반 배치
1. 관리자 흐름 (대회 세팅) — 배경 설정
2. 관전자 지도 + 실시간 추적 — 핵심 가치
3. 응원 위치 설정 + 접근 알림 — 클라이맥스
4. 관리자 대시보드 + 대회 종료 — 마무리

---

## 시나리오 1: 대회 생성과 코스 등록

**관련 유저스토리**: US-2.1, US-2.3, US-2.4
**시연 목적**: 대회 관리자가 몇 분 만에 마라톤 대회를 셋업하는 과정
**사전 조건**: 카카오 OAuth 로그인 가능한 계정, seed-course.gpx 파일
**예상 소요**: ~2분

### 동작 순서
1. `/` (랜딩 페이지)로 이동
2. "관리자 로그인" 클릭 → `/login`
3. 카카오 로그인 버튼 클릭 → OAuth 인증 → `/admin` 리다이렉트
4. "새 대회 만들기" 클릭 → `/admin/races/new`
5. 대회명: "2026 서울 마라톤" 입력
6. 대회 날짜: "2026-03-12" 입력
7. SmartChip 대회 ID: "930312" 입력
8. "대회 생성" 클릭
9. → 기대 결과: 대회 상세 페이지로 이동, 대회명과 상태(예정) 표시
10. 코스 GPX 섹션에서 seed-course.gpx 파일 업로드
11. → 기대 결과: 코스 미리보기 지도에 코스 선 표시, 총 거리 표시
12. "대회 시작 (Live)" 버튼 클릭 → 확인 다이얼로그에서 "확인"
13. → 기대 결과: 상태 뱃지가 '예정' → '진행 중'(빨간색)으로 변경

### 나레이션 포인트
- 관리자가 GPX 파일 하나로 코스 전체를 등록할 수 있다
- 대회를 'Live'로 전환하는 순간 실시간 추적이 활성화된다
- 공유 링크(`/race/930312`)가 자동 생성되어 응원자에게 바로 공유할 수 있다

## 시나리오 2: 관전자 지도에서 주자 실시간 추적

**관련 유저스토리**: US-6.1, US-6.2, US-5.1, US-5.2
**시연 목적**: 응원자가 로그인 없이 공유 링크 하나로 주자를 실시간 추적하는 핵심 경험
**사전 조건**: 시나리오 1에서 만든 대회가 live 상태, 시뮬레이션 스크립트 실행 중
**예상 소요**: ~4분 (시뮬레이션 관찰 시간 포함)

### 동작 순서
1. **[백그라운드]** 시뮬레이션 스크립트 시작 (별도 터미널):
   ```
   npx tsx scripts/simulation/runner.ts --speed 10
   ```
2. `/race/930312`로 이동 (로그인 불필요)
3. → 기대 결과: 카카오 맵에 코스 선 표시, 5km 단위 마커 표시
4. 주자 마커 5개가 코스 위에 표시됨 — 실시간으로 이동 관찰
5. → 핵심: 마커가 30초마다 점프하지 않고 **부드럽게 미끄러지듯** 이동 (2초 보간)
6. 바텀시트를 위로 드래그 → 주자 목록 확인
7. → 기대 결과: 각 주자의 배번, 이름, 현재 km, 페이스 실시간 갱신
8. 주자 목록에서 "주자 101" 탭 → 지도가 해당 주자 위치로 줌인
9. 30초 정도 관찰 — 마커 이동, km 수치 변동 확인
10. 시뮬레이션 콘솔에서 split 통과 로그 확인 (`[101] 1km 통과 - 구간: 00:04:00`)
11. 새로고침 버튼 클릭 → 최신 위치 수동 동기화

### 나레이션 포인트
- **제로 설치, 제로 로그인**: 응원자는 공유 링크만 받으면 끝. 앱 다운로드도, 회원가입도 필요 없다
- **실시간 위치 추적**: Supabase Realtime을 통해 30초 간격 위치 데이터가 모든 접속자에게 동시에 전달된다
- **부드러운 마커 애니메이션**: 30초 간격 데이터인데도 마커가 자연스럽게 이동한다 — 클라이언트 사이드 선형 보간(lerp) 기술
- **신뢰도 시각화**: Garmin GPS 데이터는 실선(high confidence), SmartChip 보간 데이터는 점선으로 구분하여 데이터의 정확도를 투명하게 보여준다
- **모바일 최적화**: 바텀시트 UI로 지도 공간을 최대한 확보하면서, 드래그 한 번으로 주자 정보에 접근

## 시나리오 3: 응원 위치 설정과 주자 접근 알림

**관련 유저스토리**: US-6.3, US-6.4, US-8.2
**시연 목적**: 응원의 핵심 가치 — "적시에, 정확한 알림"으로 응원 타이밍을 놓치지 않는 경험
**사전 조건**: 시나리오 2 상태 유지, 시뮬레이션 계속 실행 중
**예상 소요**: ~4분

### 동작 순서
1. `/race/930312` 관전자 지도 화면에서 시작
2. 응원 위치 설정 UI 접근 (CheerLocationForm)
3. "km 수동입력" 탭 선택
4. km 지점 입력: 주자들이 곧 도달할 km 지점 입력 (시뮬레이션 진행도에 따라 조정)
   - 예: 현재 주자 101이 1.5km 부근이면 → "3" 입력
5. → 기대 결과: 미리보기 지도에 해당 지점 표시, "3km 지점" 텍스트
6. 닉네임: "응원단" 입력
7. "응원 위치 설정" 버튼 클릭
8. → 기대 결과: spectators 테이블에 cheer_km=3 저장 완료
9. 시뮬레이션 관찰 — 가장 빠른 주자(101번, 4분/km)가 3km 지점에 접근 대기
10. → 기대 결과: ETA가 3분(180초) 이내가 되면 **인앱 알림 배너 팝업**
    - "주자 101 도착 예정 약 2분 30초 후"
11. 알림 배너 탭 → 지도가 해당 주자 위치로 자동 이동
12. 두 번째 주자(102번, 4분30초/km)도 곧이어 접근 → 배너 추가 (최대 3개)
13. 배너 최소화 → 우측 상단 MiniBadge로 전환

### 나레이션 포인트
- **"3분 전 알림"**: 야외에서 플래카드 들고 서 있는 응원자가 화장실 갔다가 놓치는 일이 없도록, 주자가 3분 거리에 오면 알림
- **ETA 계산**: `(응원km - 주자현재km) * 페이스(초/km)` — 단순하지만 실시간 페이스 기반이라 정확
- **30초 throttle**: 같은 주자에 대해 30초 내 중복 알림 방지 — 알림 피로 없이 적시 전달
- **GPS 자동감지 또는 수동 입력**: 야외에서 GPS로 자동 잡거나, 코스 안내도 보고 km 숫자를 입력하거나 — 두 가지 방법 모두 지원

## 시나리오 4: 관리자 실시간 대시보드와 대회 종료

**관련 유저스토리**: US-7.1, US-2.4
**시연 목적**: 관리자 관점에서의 대회 모니터링과 마무리
**사전 조건**: 시뮬레이션 계속 실행 중
**예상 소요**: ~2분

### 동작 순서
1. `/admin/races/[raceId]/dashboard`로 이동 (관리자 인증 상태)
2. → 기대 결과: 6개 통계 카드가 실시간 갱신
   - 활주 중: 5 (또는 시뮬레이션 진행에 따라 변동)
   - 완주: 0 → 점차 증가
   - DNF: 0
   - 평균 페이스: 약 5'00"/km
   - Garmin: 2
   - SmartChip: 3
3. 시뮬레이션에서 빠른 주자(101번)가 완주하는 순간 관찰
4. → 기대 결과: 활주 중 5→4, 완주 0→1 실시간 변동
5. 대회 상세 페이지로 이동 → "대회 종료" 버튼 클릭 → 확인
6. → 기대 결과: 상태가 '진행 중' → '종료'로 변경

### 나레이션 포인트
- 관리자는 하나의 대시보드에서 대회 전체 상황을 한눈에 파악
- 데이터 소스별(Garmin vs SmartChip) 주자 수로 데이터 커버리지 모니터링
- 대회 종료 시 최종 기록 확정 — 더 이상의 위치 업데이트 없음

### 스트리밍 데이터 (시뮬레이션 스크립트가 실시간 생성)

#### position_update 스트리밍 패턴
- **간격**: 30초 실시간 → `--speed 10`으로 3초 간격 재생
- **데이터 변동**:
  - currentKm: 매 스텝 +0.125km ~ +0.25km (페이스에 따라)
  - lat/lng: 코스 좌표 위를 따라 점진적 이동
  - currentPace: 일정 (각 주자 설정 페이스)
  - confidence: 'high' (시뮬레이션은 항상 high)
- **자연스러운 패턴**: 각 주자가 서로 다른 속도로 이동하므로, 시간이 지남에 따라 간격이 벌어진다

#### split_update 스트리밍 패턴
- 1km 경계 통과 시 발생
- 빠른 주자(101번)가 먼저, 느린 주자(105번)가 나중에 통과
- 각 split에 구간 시간, 누적 시간 포함

#### 시뮬레이션 속도 옵션
| 배속 | 실제 30초 → | 5km 통과 (101번 기준) | 용도 |
|------|-----------|---------------------|------|
| 1x | 30초 | ~20분 | 실제 감각 |
| 10x | 3초 | ~2분 | **데모 추천** |
| 30x | 1초 | ~40초 | 빠른 검증 |
| 60x | 0.5초 | ~20초 | 스냅샷 확인 |

## 모킹 필요 항목 및 대응

### 카카오 맵 API
- **필요**: `NEXT_PUBLIC_KAKAO_MAP_KEY`
- **없으면**: "지도 로딩 중..." 스켈레톤이 계속 표시됨
- **대응**: 키 설정 필수. 카카오 개발자 콘솔에서 JavaScript API 키 발급 필요

### 카카오 OAuth
- **필요**: Supabase Auth에 카카오 Provider 설정
- **없으면**: 로그인 불가 → 관리자 시나리오 진행 불가
- **대응**: Supabase 대시보드에서 카카오 OAuth 설정 필요. 또는 시뮬레이션 스크립트가 이미 대회+주자 데이터를 생성하므로, 관리자 시나리오(시나리오 1, 4)를 건너뛰고 관전자 시나리오(시나리오 2, 3)만 진행 가능

### Web Push
- **필요**: VAPID 키
- **없으면**: 브라우저 Push 알림 없음 (인앱 배너는 동작)
- **대응**: 인앱 배너(ApproachAlert)로 대체. 데모에서는 인앱 배너만으로 충분

### CoolSMS
- **필요**: API 키
- **대응**: SMS 시나리오 제외 (데모 범위 밖)

| 리스크 | 확률 | 대비 |
|--------|------|------|
| 카카오 맵 키 미설정 | 중 | 데모 전 반드시 확인. 없으면 지도 없이 데이터만 확인 |

| Realtime 연결 끊김 | 하 | ConnectionStatus가 재연결 상태 표시. 새로고침 버튼으로 수동 복구 |
| seed-course.gpx 경로 오류 | 하 | `supabase/seed-course.gpx` 파일 존재 확인 |
