## 7. 리스크 및 대응 계획

| 리스크 | 영향도 | 발생 가능성 | 대응 방안 |
|--------|--------|-------------|-----------|
| 이미지 업로드 성능 이슈 | 높음 | 중간 | Presigned URL 직접 업로드, 클라이언트 측 리사이징 |
| 피드 쿼리 성능 저하 | 높음 | 낮음 | 인덱스 최적화, 필요시 Redis Timeline 도입 |
| 팀원 이탈 | 높음 | 낮음 | 문서화 철저, 코드 리뷰 의무화로 지식 공유 |
| 스코프 크립 | 중간 | 높음 | MVP 기능만 엄격히 관리, 추가 요청은 백로그 |
| 보안 취약점 발견 | 높음 | 중간 | 개발 단계부터 보안 리뷰, 런칭 전 보안 점검 |
| 외부 서비스 장애 (S3, CDN) | 높음 | 낮음 | 폴백 전략 수립, 멀티 리전 고려 |
