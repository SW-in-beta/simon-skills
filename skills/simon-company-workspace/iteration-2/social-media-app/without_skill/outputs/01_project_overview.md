# 프로젝트 개요: SnapShare (인스타그램형 소셜 미디어 웹앱)

## 1. 프로젝트 비전

사용자가 사진을 공유하고, 다른 사용자를 팔로우하며, 좋아요와 댓글로 소통할 수 있는 인스타그램 스타일의 소셜 미디어 웹 애플리케이션을 구축한다.

## 2. 프로젝트 목표

- 사진 중심의 소셜 미디어 플랫폼 구축
- 직관적이고 반응형인 웹 UI/UX 제공
- 팔로우 기반 피드 시스템 구현 (최신순 정렬)
- 실시간 알림 시스템을 통한 사용자 참여 극대화
- 확장 가능한 아키텍처 설계

## 3. 핵심 기능 요약

| 기능 영역 | 설명 |
|-----------|------|
| 사용자 인증 | 회원가입, 로그인, 로그아웃, 프로필 관리 |
| 사진 업로드 | 이미지 업로드, 필터 적용, 캡션 작성 |
| 피드 | 팔로우한 사용자의 게시물을 최신순으로 표시 |
| 팔로우 | 다른 사용자 팔로우/언팔로우 |
| 좋아요 | 게시물에 좋아요 추가/취소 |
| 댓글 | 게시물에 댓글 작성/삭제 |
| 알림 | 좋아요, 댓글, 팔로우 시 알림 발송 |
| 검색/탐색 | 사용자 검색, 게시물 탐색 |

## 4. 기술 스택 (권장)

### 프론트엔드
- **프레임워크**: Next.js 14+ (App Router)
- **UI 라이브러리**: React 18+
- **스타일링**: Tailwind CSS + shadcn/ui
- **상태 관리**: Zustand 또는 React Query (서버 상태)
- **이미지 처리**: browser-image-compression (클라이언트 측 리사이징)

### 백엔드
- **런타임**: Node.js 20+
- **프레임워크**: Next.js API Routes 또는 별도 Express/Fastify 서버
- **ORM**: Prisma
- **인증**: NextAuth.js (Auth.js)

### 데이터베이스
- **주 DB**: PostgreSQL 16
- **캐시**: Redis (세션, 알림 큐)
- **파일 스토리지**: AWS S3 또는 Cloudflare R2

### 인프라
- **배포**: Vercel (프론트엔드) + AWS/Railway (백엔드)
- **CDN**: Cloudflare 또는 Vercel Edge
- **모니터링**: Sentry

## 5. 프로젝트 범위

### MVP (Phase 2에서 구현 예정)
- 사용자 인증 (이메일/비밀번호)
- 사진 업로드 및 게시물 작성
- 피드 (최신순)
- 팔로우/언팔로우
- 좋아요/댓글
- 기본 알림

### 향후 확장 (Phase 3 이후)
- 소셜 로그인 (Google, Apple)
- 스토리 기능
- DM (다이렉트 메시지)
- 해시태그 시스템
- 추천 알고리즘 기반 탐색 피드
- 리포스트/공유
- 이미지 필터

## 6. 성공 지표 (KPI)

| 지표 | 목표 |
|------|------|
| 페이지 로드 시간 | < 2초 (LCP) |
| 이미지 업로드 시간 | < 5초 (10MB 이하 기준) |
| API 응답 시간 | < 200ms (P95) |
| 테스트 커버리지 | >= 80% |
| 동시 접속자 지원 | 1,000명 |

## 7. 제약 사항 및 가정

- 웹앱만 개발 (네이티브 모바일 앱 제외)
- 동영상 업로드는 MVP에서 제외
- 피드 알고리즘은 단순 최신순 (추천 알고리즘 미적용)
- 이미지 필터는 MVP에서 제외
- 초기에는 단일 리전 배포
