## E-001: 사용자 인증

### US-001: 회원가입
> **사용자**로서, 이메일과 비밀번호로 **회원가입**할 수 있다. 그래야 앱의 기능을 이용할 수 있기 때문이다.

**인수 조건 (Acceptance Criteria):**
- [x] 이메일, 비밀번호, 사용자명, 표시 이름을 입력하여 가입할 수 있다
- [x] 이메일 형식이 유효하지 않으면 에러 메시지가 표시된다
- [x] 비밀번호가 최소 8자, 대소문자+숫자+특수문자를 포함하지 않으면 에러 메시지가 표시된다
- [x] 사용자명이 3~20자가 아니거나 영문/숫자/밑줄 외 문자를 포함하면 에러 메시지가 표시된다
- [x] 이미 사용 중인 이메일이면 "이미 사용 중인 이메일입니다" 에러가 표시된다
- [x] 이미 사용 중인 사용자명이면 "이미 사용 중인 사용자명입니다" 에러가 표시된다
- [x] 가입 성공 시 자동 로그인되어 홈 피드로 이동한다

**스토리 포인트:** 5
**우선순위:** P0

---

### US-002: 로그인
> **가입한 사용자**로서, 이메일과 비밀번호로 **로그인**할 수 있다. 그래야 내 계정에 접근할 수 있기 때문이다.

**인수 조건:**
- [x] 이메일과 비밀번호를 입력하여 로그인할 수 있다
- [x] 잘못된 이메일 또는 비밀번호일 경우 "이메일 또는 비밀번호가 올바르지 않습니다" 에러가 표시된다
- [x] 로그인 성공 시 홈 피드로 이동한다
- [x] 로그인 상태는 브라우저를 닫아도 유지된다 (refresh token)

**스토리 포인트:** 3
**우선순위:** P0

---

### US-003: 로그아웃
> **로그인한 사용자**로서, **로그아웃**할 수 있다. 그래야 다른 사람이 내 계정을 사용하는 것을 방지할 수 있기 때문이다.

**인수 조건:**
- [x] 설정 메뉴에서 로그아웃 버튼을 클릭할 수 있다
- [x] 로그아웃 시 로그인 페이지로 이동한다
- [x] 로그아웃 후 인증이 필요한 페이지 접근 시 로그인 페이지로 리다이렉트된다

**스토리 포인트:** 2
**우선순위:** P0

## E-002: 프로필 관리

### US-004: 프로필 조회
> **사용자**로서, 자신 또는 다른 사용자의 **프로필을 조회**할 수 있다. 그래야 사용자 정보와 게시물을 확인할 수 있기 때문이다.

**인수 조건:**
- [x] 프로필 사진, 표시 이름, 사용자명, 자기소개가 표시된다
- [x] 게시물 수, 팔로워 수, 팔로잉 수가 표시된다
- [x] 해당 사용자의 게시물이 그리드 형태로 표시된다
- [x] 타인의 프로필인 경우 팔로우/언팔로우 버튼이 표시된다
- [x] 자신의 프로필인 경우 "프로필 편집" 버튼이 표시된다

**스토리 포인트:** 5
**우선순위:** P0

---

### US-005: 프로필 수정
> **사용자**로서, 자신의 **프로필을 수정**할 수 있다. 그래야 최신 정보를 반영할 수 있기 때문이다.

**인수 조건:**
- [x] 프로필 사진을 변경할 수 있다 (5MB 이하, jpg/png/webp)
- [x] 표시 이름을 변경할 수 있다
- [x] 사용자명을 변경할 수 있다 (14일에 1회 제한)
- [x] 자기소개를 변경할 수 있다 (150자 이내)
- [x] 웹사이트 URL을 변경할 수 있다
- [x] 변경 사항이 즉시 반영된다

**스토리 포인트:** 5
**우선순위:** P0

## E-003: 게시물 관리

### US-006: 게시물 작성
> **사용자**로서, 사진과 캡션으로 **게시물을 작성**할 수 있다. 그래야 다른 사용자와 사진을 공유할 수 있기 때문이다.

**인수 조건:**
- [x] 갤러리에서 이미지를 선택하거나 드래그 앤 드롭으로 업로드할 수 있다
- [x] 이미지 파일은 jpg, png, webp, heic 형식만 허용된다
- [x] 이미지 크기는 10MB 이하여야 한다
- [x] 캡션을 입력할 수 있다 (최대 2,200자, 선택사항)
- [x] 위치 정보를 텍스트로 입력할 수 있다 (선택사항)
- [x] 이미지 업로드 진행률이 표시된다
- [x] 게시물 작성 후 홈 피드에서 바로 확인할 수 있다

**스토리 포인트:** 8
**우선순위:** P0

---

### US-007: 게시물 삭제
> **사용자**로서, 자신의 **게시물을 삭제**할 수 있다. 그래야 원하지 않는 게시물을 제거할 수 있기 때문이다.

**인수 조건:**
- [x] 자신의 게시물에서만 삭제 옵션이 보인다
- [x] 삭제 전 확인 대화상자가 표시된다
- [x] 삭제 시 게시물과 관련된 좋아요, 댓글이 모두 삭제된다
- [x] 삭제 후 피드에서 즉시 제거된다

**스토리 포인트:** 3
**우선순위:** P0

---

### US-008: 게시물 수정
> **사용자**로서, 자신의 **게시물 캡션을 수정**할 수 있다. 그래야 오타나 내용을 고칠 수 있기 때문이다.

**인수 조건:**
- [x] 자신의 게시물에서만 수정 옵션이 보인다
- [x] 캡션과 위치 정보만 수정 가능하다
- [x] 이미지는 변경할 수 없다
- [x] 수정 후 "수정됨" 표시가 나타난다

**스토리 포인트:** 3
**우선순위:** P1

---

### US-009: 게시물 상세 조회
> **사용자**로서, **게시물을 상세 조회**할 수 있다. 그래야 전체 캡션과 모든 댓글을 볼 수 있기 때문이다.

**인수 조건:**
- [x] 이미지가 큰 화면으로 표시된다
- [x] 전체 캡션이 표시된다
- [x] 좋아요 수와 모든 댓글이 표시된다
- [x] 좋아요 버튼과 댓글 입력란이 있다
- [x] 작성 시간이 절대 시간으로 표시된다

**스토리 포인트:** 5
**우선순위:** P0

## E-004: 피드

### US-010: 홈 피드 조회
> **사용자**로서, 팔로우한 사용자의 게시물을 **최신순으로 볼** 수 있다. 그래야 관심 있는 사람들의 최신 소식을 확인할 수 있기 때문이다.

**인수 조건:**
- [x] 팔로우한 사용자의 게시물이 최신순으로 표시된다
- [x] 자신의 게시물도 피드에 표시된다
- [x] 각 게시물에 작성자 정보, 이미지, 캡션, 좋아요 수, 댓글 수가 표시된다
- [x] 아래로 스크롤하면 추가 게시물이 자동 로딩된다 (무한 스크롤)
- [x] 위로 당기면 새 게시물을 새로고침할 수 있다 (pull-to-refresh)
- [x] 팔로우한 사용자가 없으면 "사용자를 팔로우해보세요" 안내가 표시된다

**스토리 포인트:** 8
**우선순위:** P0

---

### US-011: 탐색 피드 조회
> **사용자**로서, **인기 게시물을 탐색**할 수 있다. 그래야 새로운 사용자와 콘텐츠를 발견할 수 있기 때문이다.

**인수 조건:**
- [x] 최근 24시간 내 인기 게시물이 3열 그리드로 표시된다
- [x] 게시물 클릭 시 상세 페이지로 이동한다
- [x] 무한 스크롤이 동작한다

**스토리 포인트:** 5
**우선순위:** P2

---

## E-005: 팔로우

### US-012: 팔로우하기
> **사용자**로서, 다른 사용자를 **팔로우**할 수 있다. 그래야 해당 사용자의 게시물을 피드에서 볼 수 있기 때문이다.

**인수 조건:**
- [x] 타인의 프로필에서 "팔로우" 버튼을 클릭하여 팔로우할 수 있다
- [x] 팔로우 후 버튼이 "팔로잉"으로 변경된다
- [x] 팔로우한 사용자의 게시물이 홈 피드에 나타난다
- [x] 상대방에게 팔로우 알림이 발송된다

**스토리 포인트:** 3
**우선순위:** P0

---

### US-013: 언팔로우하기
> **사용자**로서, 팔로우 중인 사용자를 **언팔로우**할 수 있다. 그래야 해당 사용자의 게시물이 피드에서 제외되기 때문이다.

**인수 조건:**
- [x] "팔로잉" 버튼을 클릭하면 확인 대화상자가 표시된다
- [x] 언팔로우 후 버튼이 "팔로우"로 변경된다
- [x] 언팔로우한 사용자의 게시물이 피드에서 제거된다
- [x] 언팔로우 시 알림은 발송되지 않는다

**스토리 포인트:** 2
**우선순위:** P0

---

### US-014: 팔로워/팔로잉 목록 조회
> **사용자**로서, **팔로워 및 팔로잉 목록을 조회**할 수 있다. 그래야 어떤 사용자와 연결되어 있는지 확인할 수 있기 때문이다.

**인수 조건:**
- [x] 프로필 페이지에서 팔로워 수를 클릭하면 팔로워 목록이 표시된다
- [x] 프로필 페이지에서 팔로잉 수를 클릭하면 팔로잉 목록이 표시된다
- [x] 각 사용자 옆에 팔로우/언팔로우 버튼이 표시된다
- [x] 목록은 스크롤로 추가 로딩할 수 있다

**스토리 포인트:** 5
**우선순위:** P0

## E-006: 좋아요

### US-015: 좋아요 추가
> **사용자**로서, 게시물에 **좋아요**를 할 수 있다. 그래야 마음에 드는 게시물에 반응을 보낼 수 있기 때문이다.

**인수 조건:**
- [x] 하트 아이콘을 클릭하여 좋아요를 추가할 수 있다
- [x] 이미지를 더블 클릭하여 좋아요를 추가할 수 있다
- [x] 좋아요 추가 시 하트 아이콘이 채워진 상태로 변경되고 좋아요 수가 증가한다
- [x] 좋아요 시 하트 애니메이션이 표시된다
- [x] 게시물 작성자에게 좋아요 알림이 발송된다

**스토리 포인트:** 3
**우선순위:** P0

---

### US-016: 좋아요 취소
> **사용자**로서, 좋아요를 **취소**할 수 있다. 그래야 실수로 누른 좋아요를 되돌릴 수 있기 때문이다.

**인수 조건:**
- [x] 채워진 하트 아이콘을 클릭하면 좋아요가 취소된다
- [x] 좋아요 취소 시 하트가 빈 상태로 변경되고 좋아요 수가 감소한다

**스토리 포인트:** 1
**우선순위:** P0

---

### US-017: 좋아요한 사용자 목록 조회
> **사용자**로서, 게시물에 **좋아요한 사용자 목록**을 볼 수 있다. 그래야 누가 관심을 보였는지 알 수 있기 때문이다.

**인수 조건:**
- [x] 좋아요 수를 클릭하면 좋아요한 사용자 목록이 표시된다
- [x] 각 사용자 옆에 팔로우/언팔로우 버튼이 표시된다

**스토리 포인트:** 3
**우선순위:** P1

## E-007: 댓글

### US-018: 댓글 작성
> **사용자**로서, 게시물에 **댓글**을 달 수 있다. 그래야 게시물에 대한 의견을 남길 수 있기 때문이다.

**인수 조건:**
- [x] 댓글 입력란에 텍스트를 입력하고 "게시" 버튼을 클릭하면 댓글이 작성된다
- [x] 댓글은 최대 1,000자까지 입력할 수 있다
- [x] 빈 댓글은 작성할 수 없다 ("게시" 버튼 비활성화)
- [x] 댓글 작성 후 댓글 목록에 즉시 표시된다
- [x] 게시물 작성자에게 댓글 알림이 발송된다

**스토리 포인트:** 3
**우선순위:** P0

---

### US-019: 댓글 삭제
> **사용자**로서, 자신의 **댓글을 삭제**할 수 있다. 그래야 잘못된 댓글을 제거할 수 있기 때문이다.

**인수 조건:**
- [x] 자신의 댓글을 길게 누르거나 우클릭하면 삭제 옵션이 나타난다
- [x] 게시물 작성자는 자신의 게시물에 달린 모든 댓글을 삭제할 수 있다
- [x] 삭제 시 댓글 목록에서 즉시 제거된다

**스토리 포인트:** 2
**우선순위:** P0

---

### US-020: 댓글 목록 조회
> **사용자**로서, 게시물의 **전체 댓글을 조회**할 수 있다. 그래야 다른 사람들의 의견을 확인할 수 있기 때문이다.

**인수 조건:**
- [x] 피드에서 "댓글 N개 모두 보기"를 클릭하면 전체 댓글이 표시된다
- [x] 댓글은 작성 시간 오름차순으로 정렬된다
- [x] 스크롤로 과거 댓글을 추가 로딩할 수 있다

**스토리 포인트:** 3
**우선순위:** P0

---

## E-008: 알림

### US-021: 알림 목록 조회
> **사용자**로서, **알림 목록을 조회**할 수 있다. 그래야 나에게 일어난 활동을 확인할 수 있기 때문이다.

**인수 조건:**
- [x] 좋아요, 댓글, 팔로우 알림이 최신순으로 표시된다
- [x] 각 알림에 행위자 프로필 사진, 내용, 시간이 표시된다
- [x] 좋아요/댓글 알림 클릭 시 해당 게시물로 이동한다
- [x] 팔로우 알림 클릭 시 해당 사용자 프로필로 이동한다
- [x] 읽지 않은 알림은 시각적으로 구별된다

**스토리 포인트:** 5
**우선순위:** P1

### US-022: 알림 배지
> **사용자**로서, **읽지 않은 알림 수를 배지로** 볼 수 있다. 그래야 새로운 활동이 있는지 빠르게 알 수 있기 때문이다.

**인수 조건:**
- [x] 하단/상단 네비게이션의 알림 아이콘에 읽지 않은 알림 수가 배지로 표시된다
- [x] 알림 페이지 진입 시 모든 알림이 읽음 처리된다
- [x] 배지 수는 30초마다 자동 갱신된다

**스토리 포인트:** 3
**우선순위:** P1

## E-009: 검색

### US-023: 사용자 검색
> **사용자**로서, **다른 사용자를 검색**할 수 있다. 그래야 관심 있는 사용자를 찾아 팔로우할 수 있기 때문이다.

**인수 조건:**
- [x] 검색창에 텍스트 입력 시 실시간으로 결과가 표시된다 (debounce 300ms)
- [x] 사용자명과 표시 이름에서 검색된다
- [x] 검색 결과에 프로필 사진, 사용자명, 표시 이름이 표시된다
- [x] 검색 결과 클릭 시 해당 사용자 프로필로 이동한다

**스토리 포인트:** 5
**우선순위:** P1
