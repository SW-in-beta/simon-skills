# 기능 명세서 (Feature Specifications)

## 1. 사용자 인증 (Authentication)

### 1.1 회원가입
- **설명**: 새로운 사용자가 이메일과 비밀번호로 계정을 생성한다.
- **입력**:
  - 이메일 (필수, 유효한 이메일 형식)
  - 비밀번호 (필수, 최소 8자, 대소문자+숫자+특수문자 포함)
  - 사용자명 (필수, 3~20자, 영문/숫자/밑줄만 허용, 고유)
  - 표시 이름 (필수, 1~50자)
- **처리**:
  - 이메일 중복 확인
  - 사용자명 중복 확인
  - 비밀번호 해싱 (bcrypt, cost factor 12)
  - 인증 이메일 발송 (선택, MVP 후순위)
- **출력**: JWT 토큰 (access + refresh)
- **에러 케이스**:
  - 이미 존재하는 이메일 (409)
  - 이미 존재하는 사용자명 (409)
  - 비밀번호 정책 미달 (400)
  - 잘못된 이메일 형식 (400)

### 1.2 로그인
- **설명**: 기존 사용자가 이메일과 비밀번호로 로그인한다.
- **입력**: 이메일, 비밀번호
- **처리**:
  - 이메일로 사용자 조회
  - 비밀번호 검증
  - JWT 토큰 발급 (access: 15분, refresh: 7일)
- **출력**: JWT 토큰 + 사용자 기본 정보
- **에러 케이스**:
  - 존재하지 않는 이메일 (401)
  - 잘못된 비밀번호 (401)
  - 비활성 계정 (403)

### 1.3 로그아웃
- **설명**: 현재 세션을 종료한다.
- **처리**: refresh 토큰 블랙리스트 등록 (Redis)
- **출력**: 성공 메시지

### 1.4 토큰 갱신
- **설명**: 만료된 access 토큰을 refresh 토큰으로 갱신한다.
- **입력**: refresh 토큰
- **출력**: 새로운 access 토큰

## 2. 프로필 관리 (Profile)

### 2.1 프로필 조회
- **설명**: 자신 또는 다른 사용자의 프로필을 조회한다.
- **출력**:
  - 프로필 사진 URL
  - 표시 이름
  - 사용자명
  - 자기소개 (bio, 최대 150자)
  - 웹사이트 URL (선택)
  - 게시물 수
  - 팔로워 수
  - 팔로잉 수
  - 팔로우 여부 (타인 프로필인 경우)
  - 최근 게시물 목록 (그리드 뷰, 페이지네이션)

### 2.2 프로필 수정
- **설명**: 자신의 프로필 정보를 수정한다.
- **수정 가능 필드**:
  - 프로필 사진 (이미지 업로드, 최대 5MB, jpg/png/webp)
  - 표시 이름
  - 사용자명 (중복 확인)
  - 자기소개 (bio)
  - 웹사이트 URL
- **제약사항**:
  - 사용자명 변경은 14일에 1회 제한

## 3. 사진 업로드 및 게시물 (Posts)

### 3.1 게시물 작성
- **설명**: 사진과 함께 새 게시물을 작성한다.
- **입력**:
  - 이미지 파일 (필수, 최대 10MB, jpg/png/webp/heic)
  - 캡션 (선택, 최대 2,200자)
  - 위치 정보 (선택, 텍스트)
- **처리**:
  - 이미지 유효성 검증 (형식, 크기, EXIF 데이터 제거)
  - 클라이언트 측 이미지 리사이징 (최대 1080px 너비)
  - S3 업로드 후 CDN URL 생성
  - 썸네일 생성 (300x300, 600x600)
- **출력**: 생성된 게시물 정보 (ID, 이미지 URL, 캡션, 생성시간)
- **제약사항**:
  - MVP에서는 게시물당 이미지 1장
  - 향후 다중 이미지 (캐러셀) 지원 예정

### 3.2 게시물 조회
- **설명**: 특정 게시물의 상세 정보를 조회한다.
- **출력**:
  - 이미지 URL
  - 캡션
  - 작성자 정보 (사용자명, 프로필 사진)
  - 좋아요 수
  - 댓글 목록 (최신 3개 + 더보기)
  - 좋아요 여부
  - 작성 시간 (상대 시간 표시)
  - 위치 정보

### 3.3 게시물 삭제
- **설명**: 자신의 게시물을 삭제한다.
- **처리**:
  - 게시물 소프트 삭제 (DB)
  - 연관된 좋아요, 댓글 소프트 삭제
  - S3 이미지 삭제는 배치 작업으로 비동기 처리
- **제약사항**: 본인의 게시물만 삭제 가능

### 3.4 게시물 수정
- **설명**: 자신의 게시물 캡션을 수정한다.
- **수정 가능 필드**: 캡션, 위치 정보
- **제약사항**: 이미지 변경 불가

## 4. 피드 (Feed)

### 4.1 홈 피드
- **설명**: 팔로우한 사용자들의 게시물을 최신순으로 표시한다.
- **알고리즘**: 단순 최신순 (created_at DESC)
- **페이지네이션**: 커서 기반 (cursor-based pagination)
  - 한 페이지당 10개 게시물
  - 무한 스크롤 방식
- **표시 정보** (각 게시물):
  - 작성자 프로필 사진 + 사용자명
  - 이미지
  - 좋아요 수
  - 캡션 (접혀 있는 상태, 더보기 클릭 시 전체 표시)
  - 댓글 수 + 최신 댓글 2개
  - 작성 시간 (상대 시간)
  - 좋아요 버튼, 댓글 버튼
- **최적화**:
  - 이미지 lazy loading
  - 피드 데이터 캐싱 (React Query)
  - 이미지 srcset을 활용한 반응형 이미지

### 4.2 탐색 피드
- **설명**: 팔로우하지 않은 사용자의 인기 게시물을 표시한다.
- **알고리즘**: 최근 24시간 내 좋아요 수 기준 내림차순
- **표시 형태**: 그리드 레이아웃 (3열)
- **페이지네이션**: 커서 기반, 무한 스크롤

---

## 5. 팔로우 시스템 (Follow)

### 5.1 팔로우
- **설명**: 다른 사용자를 팔로우한다.
- **처리**:
  - 팔로우 관계 생성
  - 상대방에게 알림 발송
  - 팔로워/팔로잉 카운트 갱신
- **제약사항**:
  - 자기 자신 팔로우 불가
  - 이미 팔로우 중인 경우 중복 요청 무시 (멱등성)

### 5.2 언팔로우
- **설명**: 팔로우 중인 사용자를 언팔로우한다.
- **처리**:
  - 팔로우 관계 삭제
  - 팔로워/팔로잉 카운트 갱신
  - 알림은 발송하지 않음

### 5.3 팔로워 목록 조회
- **설명**: 특정 사용자의 팔로워 목록을 조회한다.
- **출력**: 팔로워 목록 (프로필 사진, 사용자명, 표시 이름, 팔로우 여부)
- **페이지네이션**: 커서 기반, 20명씩

### 5.4 팔로잉 목록 조회
- **설명**: 특정 사용자가 팔로우하는 사용자 목록을 조회한다.
- **출력**: 팔로잉 목록 (프로필 사진, 사용자명, 표시 이름, 팔로우 여부)
- **페이지네이션**: 커서 기반, 20명씩

## 6. 좋아요 (Like)

### 6.1 좋아요 추가
- **설명**: 게시물에 좋아요를 추가한다.
- **처리**:
  - 좋아요 레코드 생성
  - 게시물 좋아요 카운트 증가
  - 게시물 작성자에게 알림 발송 (본인 게시물 제외)
- **제약사항**: 중복 좋아요 불가 (멱등성)

### 6.2 좋아요 취소
- **설명**: 게시물의 좋아요를 취소한다.
- **처리**:
  - 좋아요 레코드 삭제
  - 게시물 좋아요 카운트 감소

### 6.3 좋아요한 사용자 목록
- **설명**: 특정 게시물에 좋아요한 사용자 목록을 조회한다.
- **출력**: 사용자 목록 (프로필 사진, 사용자명, 팔로우 여부)
- **페이지네이션**: 커서 기반, 20명씩

---

## 7. 댓글 (Comment)

### 7.1 댓글 작성
- **설명**: 게시물에 댓글을 작성한다.
- **입력**: 댓글 내용 (필수, 최대 1,000자)
- **처리**:
  - 댓글 레코드 생성
  - 게시물 댓글 카운트 증가
  - 게시물 작성자에게 알림 발송 (본인 게시물 제외)
- **제약사항**: 빈 댓글 불가

### 7.2 댓글 삭제
- **설명**: 자신의 댓글 또는 자신의 게시물에 달린 댓글을 삭제한다.
- **처리**:
  - 댓글 소프트 삭제
  - 게시물 댓글 카운트 감소

### 7.3 댓글 목록 조회
- **설명**: 특정 게시물의 댓글 목록을 조회한다.
- **정렬**: 작성 시간 오름차순 (오래된 것부터)
- **출력**: 댓글 목록 (작성자 정보, 댓글 내용, 작성 시간)
- **페이지네이션**: 커서 기반, 20개씩

## 8. 알림 (Notification)

### 8.1 알림 유형
| 유형 | 트리거 | 메시지 예시 |
|------|--------|-------------|
| 좋아요 | 게시물에 좋아요 추가 | "{사용자명}님이 회원님의 게시물을 좋아합니다" |
| 댓글 | 게시물에 댓글 작성 | "{사용자명}님이 댓글을 남겼습니다: {댓글 미리보기}" |
| 팔로우 | 사용자 팔로우 | "{사용자명}님이 회원님을 팔로우합니다" |

### 8.2 알림 목록 조회
- **설명**: 사용자의 알림 목록을 조회한다.
- **출력**: 알림 목록 (유형, 행위자 정보, 대상, 시간, 읽음 여부)
- **정렬**: 최신순
- **페이지네이션**: 커서 기반, 20개씩

### 8.3 알림 읽음 처리
- **설명**: 알림을 읽음 상태로 변경한다.
- **처리**:
  - 개별 알림 읽음 처리
  - 전체 알림 일괄 읽음 처리

### 8.4 읽지 않은 알림 수 조회
- **설명**: 읽지 않은 알림의 개수를 조회한다.
- **활용**: 앱 헤더의 알림 배지에 표시
- **최적화**: Redis 카운터 활용

### 8.5 실시간 알림 (향후)
- WebSocket 또는 SSE를 통한 실시간 알림 푸시
- MVP에서는 폴링 방식 (30초 간격)으로 구현

## 9. 검색 (Search)

### 9.1 사용자 검색
- **설명**: 사용자명 또는 표시 이름으로 사용자를 검색한다.
- **입력**: 검색어 (최소 1자)
- **처리**: 사용자명, 표시 이름에서 prefix 매칭
- **출력**: 사용자 목록 (프로필 사진, 사용자명, 표시 이름, 팔로우 여부)
- **제약사항**: 최대 50개 결과

## 10. 비기능 요구사항

### 10.1 보안
- 모든 API에 인증 필수 (public 엔드포인트 제외)
- Rate limiting: 인증 API 5회/분, 일반 API 100회/분
- 이미지 업로드 시 악성 파일 검사
- SQL Injection 방지 (ORM 사용)
- XSS 방지 (입력값 sanitize)
- CORS 설정

### 10.2 성능
- 이미지 최적화: WebP 변환, 다중 해상도 제공
- 피드 API 응답: < 200ms (P95)
- 이미지 업로드: < 5초 (10MB 기준)
- 무한 스크롤 smooth 경험

### 10.3 접근성
- WCAG 2.1 AA 준수
- 키보드 네비게이션 지원
- 스크린 리더 호환 (alt 텍스트, ARIA 레이블)
- 다크 모드 지원

### 10.4 반응형 디자인
- 모바일 (320px~767px)
- 태블릿 (768px~1023px)
- 데스크톱 (1024px~)
