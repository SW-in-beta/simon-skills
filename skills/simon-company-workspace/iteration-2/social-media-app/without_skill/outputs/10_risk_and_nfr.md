## 2. 비기능 요구사항 (NFR)

### 2.1 성능 (Performance)

| ID | 요구사항 | 목표 | 측정 방법 |
|----|----------|------|-----------|
| NFR-P01 | 페이지 초기 로딩 시간 (LCP) | < 2.0초 | Lighthouse, Web Vitals |
| NFR-P02 | API 응답 시간 (P50) | < 100ms | APM 모니터링 |
| NFR-P03 | API 응답 시간 (P95) | < 200ms | APM 모니터링 |
| NFR-P04 | API 응답 시간 (P99) | < 500ms | APM 모니터링 |
| NFR-P05 | 이미지 업로드 완료 시간 | < 5초 (10MB) | 클라이언트 측 측정 |
| NFR-P06 | 피드 스크롤 프레임률 | >= 60fps | Chrome DevTools |
| NFR-P07 | First Input Delay (FID) | < 100ms | Web Vitals |
| NFR-P08 | Cumulative Layout Shift (CLS) | < 0.1 | Web Vitals |
| NFR-P09 | Time to First Byte (TTFB) | < 200ms | Web Vitals |
| NFR-P10 | 번들 크기 (JS) | < 200KB (gzip) | webpack-bundle-analyzer |

## 3. 기술 부채 관리 계획

| 항목 | 현재 상태 (MVP) | 향후 개선 |
|------|-----------------|-----------|
| 알림 방식 | 30초 간격 폴링 | WebSocket 또는 SSE로 실시간화 |
| 피드 알고리즘 | 단순 최신순 SQL | Redis Timeline + 추천 알고리즘 |
| 이미지 처리 | 클라이언트 리사이징 | 서버 측 Lambda로 다중 해상도 생성 |
| 검색 | PostgreSQL LIKE | Elasticsearch 또는 Meilisearch |
| 캐싱 | React Query만 | Redis 캐시 레이어 추가 |
| 모니터링 | 기본 로깅 | 분산 추적 (OpenTelemetry) |
