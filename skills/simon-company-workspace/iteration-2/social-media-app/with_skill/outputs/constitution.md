# Constitution: Picstory - 소셜 미디어 웹앱

## Core Principles

1. **사용자 경험 최우선**: 빠른 로딩, 직관적 UI, 부드러운 인터랙션. 기능보다 UX를 우선한다.
2. **보안은 타협 불가**: 인증/인가, 비밀번호 해싱, SQL Injection 방지, XSS 방지. OWASP Top 10 준수.
3. **단순함 우선 (KISS)**: 피드 알고리즘은 최신순. 불필요한 복잡성을 도입하지 않는다.
4. **점진적 가치 전달**: Walking Skeleton부터 동작하는 소프트웨어를 만들고, 점진적으로 기능을 추가한다.
5. **테스트 주도 개발**: TDD 사이클(Red-Green-Refactor)을 준수한다. 테스트 없는 코드는 작성하지 않는다.

## Quality Gates

- 테스트 커버리지: 80% 이상
- TRP 3라운드 통과 필수
- OWASP Top 10 준수
- 빌드 성공 + 린트 통과 필수
- API 응답 시간: P95 < 500ms
- 이미지 업로드 처리: < 5초
- 피드 로딩: < 2초 (첫 10개 게시물)

## Constraints

### 기술적 제약
- 웹앱으로 구현 (모바일 앱 아님)
- 피드 알고리즘은 단순 최신순 (ML/추천 알고리즘 없음)
- 이미지 파일 형식: JPG, PNG, GIF
- 이미지 최대 크기: 10MB
- 캡션 최대 길이: 2200자
- 댓글 최대 길이: 1000자

### 비즈니스 제약
- MVP 범위: P1 스토리 전체 (13개 중 9개)
- 실시간 메시징/DM은 범위 밖
- 스토리/릴스 기능은 범위 밖
- 소셜 로그인(Google, Facebook)은 범위 밖 (이메일/비밀번호만)

### 비기능 요구사항
- 동시 접속 100명 이상 지원
- 반응형 웹 (모바일/태블릿/데스크톱)
- WCAG 2.1 AA 접근성 준수
- 한국어 기본, 다국어 지원은 범위 밖
