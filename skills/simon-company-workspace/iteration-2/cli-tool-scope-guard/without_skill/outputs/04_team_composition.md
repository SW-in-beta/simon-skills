# 팀 구성 및 역할 정의

## 1. 프로젝트 규모 평가

본 프로젝트는 소규모 CLI 도구 개발로, 1~2명의 개발자가 약 8일(2주 이내) 안에 완료 가능한 규모이다.

### 복잡도 분석

| 영역 | 복잡도 | 근거 |
|------|--------|------|
| CSV 파싱 | 낮음 | Go 표준 라이브러리 활용 |
| JSON 출력 | 낮음 | Go 표준 라이브러리 활용 |
| 인코딩 감지 | 중간 | 외부 라이브러리 활용하나 엣지 케이스 다수 |
| 컬럼 필터링 | 낮음 | 단순 인덱스 기반 필터링 |
| 스트리밍 처리 | 중간 | io.Reader 체이닝 설계 필요 |
| CLI 인터페이스 | 낮음 | cobra 프레임워크 활용 |
| 크로스 플랫폼 빌드 | 낮음 | GoReleaser 활용 |

## 2. 권장 팀 구성

### 옵션 A: 1인 개발 (권장)

| 역할 | 담당자 | 책임 |
|------|--------|------|
| 개발자 (풀스택) | 1명 | 설계, 구현, 테스트, CI/CD, 배포 |

**적합한 경우:**
- Go 경험이 있는 시니어 개발자
- CLI 도구 개발 경험 보유
- 8일 풀타임 투입 가능

### 옵션 B: 2인 개발

| 역할 | 담당자 | 책임 |
|------|--------|------|
| 리드 개발자 | 1명 | 아키텍처 설계, 코어 파이프라인, 코드 리뷰 |
| 개발자 | 1명 | CLI 인터페이스, 테스트 작성, CI/CD 구성 |

**적합한 경우:**
- 주니어 개발자의 성장 기회로 활용
- 더 빠른 완료가 필요한 경우 (5일 목표)

## 3. 필요 역량

### 필수 역량
- Go 언어 중급 이상 (인터페이스, io.Reader/Writer 패턴 이해)
- CLI 도구 개발 경험 (플래그 파싱, stdin/stdout 처리)
- 유닛 테스트 작성 능력 (TDD 경험 우대)
- Git/GitHub 워크플로우

### 우대 역량
- 문자 인코딩 지식 (UTF-8, EUC-KR, Shift-JIS 등)
- 스트리밍 데이터 처리 경험
- GoReleaser, GitHub Actions 경험
- RFC 4180 (CSV 형식) 이해

## 4. 외부 리뷰어

| 역할 | 시점 | 목적 |
|------|------|------|
| 코드 리뷰어 | PR 단위 | 코드 품질, 설계 검토 |
| QA (수동) | M5 완료 후 | 다양한 인코딩 파일로 실사용 테스트 |
