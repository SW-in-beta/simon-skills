# 범위 정의서 (Scope Definition)

## 1. 범위 내 (In Scope)

### 1.1 핵심 기능 (Must Have)

#### CSV 파싱
- RFC 4180 호환 CSV 파싱
- 구분자 자동 감지 (쉼표, 탭, 세미콜론, 파이프)
- 구분자 수동 지정 옵션 (`--delimiter`)
- 따옴표 내 줄바꿈, 이스케이프 문자 처리
- 헤더 행 유/무 옵션 (`--no-header`)

#### JSON 출력
- 배열 형태 JSON 출력 (기본값)
- NDJSON (Newline Delimited JSON) 출력 (`--format ndjson`)
- stdout 출력 (기본값) 및 파일 출력 (`-o`, `--output`)
- 들여쓰기 옵션 (`--pretty`, `--indent`)

#### 인코딩 자동 감지
- 파일 시작 부분 샘플링을 통한 인코딩 자동 판별
- 지원 인코딩: UTF-8, UTF-8 BOM, UTF-16 LE/BE, EUC-KR, Shift-JIS, ISO-8859-1, CP949
- 인코딩 수동 지정 옵션 (`--encoding`)
- 감지된 인코딩 정보 표시 (`--verbose`)

#### 컬럼 필터링
- 컬럼 이름 기반 필터링 (`--columns "name,email"`)
- 컬럼 인덱스 기반 필터링 (`--columns "0,2,5"`)
- 컬럼 제외 필터링 (`--exclude-columns "password,secret"`)

#### 대용량 파일 스트리밍
- 스트리밍 모드 (`--stream`): 행 단위 처리로 메모리 사용량 최소화
- stdin 입력 지원 (파이프라인 호환)
- 진행률 표시 (`--progress`)

### 1.2 부가 기능 (Should Have)
- `--version` 버전 정보 출력
- `--help` 도움말 출력
- 에러 발생 시 명확한 에러 메시지 및 종료 코드
- 빈 값 처리 옵션 (`--null-value`): 빈 문자열을 null로 변환 여부
- 숫자 자동 변환 옵션 (`--type-infer`): 문자열 "123"을 숫자 123으로 변환

### 1.3 비기능 요구사항
- 1GB 이상 파일 처리 시 메모리 사용량 100MB 이하 (스트리밍 모드)
- 단일 바이너리 배포 (크로스 플랫폼: Linux, macOS, Windows)
- 테스트 커버리지 80% 이상
- CI/CD 파이프라인 구성 (빌드, 테스트, 릴리즈)

## 2. 범위 외 (Out of Scope)

| 항목 | 사유 |
|------|------|
| JSON-to-CSV 역변환 | 별도 도구로 분리, 향후 확장 고려 |
| GUI 인터페이스 | CLI 도구로 한정 |
| 데이터 유효성 검증 (스키마 기반) | JSON Schema 검증은 별도 도구 영역 |
| 원격 파일 다운로드 (HTTP/S3) | curl 등 외부 도구와 파이프로 조합 |
| Excel (.xlsx) 파일 지원 | CSV만 대상, Excel은 별도 도구 |
| 데이터 변환/매핑 (컬럼명 변경, 값 변환) | jq 등 후처리 도구와 조합 |
| 병렬 처리 (멀티스레드) | v1.0에서는 단일 스레드, 향후 확장 |

## 3. 가정 사항 (Assumptions)
- 입력 파일은 로컬 파일시스템에 존재하거나 stdin으로 전달된다
- CSV 파일의 각 행은 동일한 수의 컬럼을 가진다 (불일치 시 에러 또는 경고)
- 인코딩 자동 감지의 정확도는 100%가 아닐 수 있으며, 수동 지정으로 보완한다
- 출력 인코딩은 항상 UTF-8이다

| M4: 스트리밍 | 대용량 파일 스트리밍 + NDJSON | 2일 |
