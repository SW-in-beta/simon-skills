# 프로젝트 개요: 사용자 행동 기반 상품 추천 시스템

## 1. 프로젝트명
**RecoPilot** - 사용자 행동 기반 협업 필터링 상품 추천 시스템

## 2. 프로젝트 배경
사용자에게 개인화된 상품 추천을 제공하여 구매 전환율과 사용자 경험을 향상시키고자 한다.
기존에는 규칙 기반(인기 상품, 카테고리 기반) 추천만 제공하고 있어,
사용자 개별 행동 패턴을 반영한 정교한 추천이 부재한 상황이다.

## 3. 프로젝트 목표
- 사용자의 행동 데이터(조회, 클릭, 구매, 장바구니 등)를 기반으로 협업 필터링(Collaborative Filtering) 모델을 구축한다.
- REST API를 통해 실시간 추천 결과를 서비스에 제공한다.
- 관리자 대시보드를 통해 추천 성능 지표(CTR, 전환율, 다양성 등)를 모니터링할 수 있게 한다.

## 4. 핵심 가치
| 가치 | 설명 |
|------|------|
| 개인화 | 사용자마다 다른 맞춤 추천 제공 |
| 실시간성 | API 호출 시 200ms 이내 응답 |
| 측정 가능성 | 추천 성능을 정량적으로 모니터링 |
| 확장성 | 추후 딥러닝 모델, 하이브리드 모델로 확장 가능한 구조 |

## 5. 프로젝트 범위

### In-Scope
- 협업 필터링 기반 추천 모델 (User-Based CF, Item-Based CF, Matrix Factorization)
- 추천 결과 제공 REST API
- 관리자 대시보드 (추천 성능 모니터링)
- 사용자 행동 데이터 수집 파이프라인
- A/B 테스트 프레임워크 (기본)

### Out-of-Scope (향후 Phase에서 고려)
- 콘텐츠 기반 필터링 (Content-Based Filtering)
- 딥러닝 기반 추천 모델 (NCF, Transformer 등)
- 실시간 스트리밍 기반 모델 업데이트
- 추천 사유 설명 (Explainability)
- 모바일 전용 SDK

## 6. 성공 지표 (KPI)
| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 추천 클릭률 (CTR) | 기존 대비 20% 향상 | 추천 노출 대비 클릭 수 |
| 구매 전환율 | 기존 대비 15% 향상 | 추천 클릭 대비 구매 수 |
| API 응답 시간 (P99) | 200ms 이하 | API 모니터링 |
| 추천 커버리지 | 80% 이상 | 추천 가능한 상품 비율 |
| 추천 다양성 | Gini 계수 0.5 이하 | 추천 상품 분포 |

## 7. 이해관계자
| 역할 | 책임 |

| ML Engineer | 추천 모델 개발 및 학습 파이프라인 구축 |
| Backend Engineer | REST API 개발, 데이터 파이프라인 구축 |
| Frontend Engineer | 관리자 대시보드 개발 |
| Data Engineer | 데이터 인프라 및 ETL 파이프라인 |

## 8. 주요 마일스톤
| 마일스톤 | 예상 기간 | 설명 |
|----------|-----------|------|
| Phase 1: 기획 및 설계 | 2주 | 요구사항 분석, 아키텍처 설계, 기술 스택 결정 |
| Phase 2: 데이터 파이프라인 | 3주 | 행동 데이터 수집, 전처리, 저장소 구축 |
| Phase 3: 모델 개발 | 4주 | CF 모델 개발, 학습, 평가 |
| Phase 4: API 개발 | 3주 | 추천 API 개발, 캐싱, 배포 |
| Phase 5: 대시보드 | 3주 | 관리자 모니터링 대시보드 |
| Phase 6: 통합 테스트 및 출시 | 2주 | E2E 테스트, A/B 테스트, 점진적 출시 |
