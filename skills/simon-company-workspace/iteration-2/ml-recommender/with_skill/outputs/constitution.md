# Constitution: 사용자 행동 기반 상품 추천 시스템

**Created**: 2026-03-07
**Status**: Approved

## Core Principles

1. **추천 품질 최우선** — 추천 결과의 정확성과 관련성이 시스템의 존재 이유이다. 성능 최적화나 기능 확장보다 추천 품질을 우선한다.
2. **데이터 무결성 보장** — 사용자 행동 데이터는 추천의 근간이다. 데이터 유실, 오염, 중복을 방지하는 데 최선을 다한다.
3. **API 안정성과 신뢰성** — 외부 서비스가 의존하는 REST API는 항상 일관된 응답 형식과 합리적인 응답 시간을 보장해야 한다.
4. **측정 가능한 개선** — 추천 시스템의 모든 변경은 성능 지표(CTR, 전환율, 커버리지)로 측정하고, 데이터에 기반한 의사결정을 한다.
5. **점진적 복잡도 관리** — 단순한 모델부터 시작하여 검증된 후에 복잡도를 높인다. 과잉 엔지니어링을 지양한다.

## Quality Gates

- 테스트 커버리지: 80% 이상
- TRP 3라운드 통과 필수
- 추천 API 응답 시간: 95th percentile 200ms 이내
- Cold Start 사용자에게도 최소 5개 추천 결과 제공
- 모든 API 엔드포인트에 입력 유효성 검증 필수
- 보안: API Key 인증 필수, SQL Injection/XSS 방어
- 모델 학습 실패 시 이전 모델 자동 폴백

## Constraints

### 기술적 제약
- Collaborative Filtering 알고리즘 사용 (User-based 또는 Item-based, 또는 Matrix Factorization)
- REST API 형식 (JSON)
- 관리자 대시보드는 웹 기반
- 모델 학습은 배치 방식 (실시간 학습은 P3 이후)

### 비즈니스 제약
- 사용자 개인정보는 최소한만 저장 (user_id만, 이름/이메일 등 미저장)
- 추천 결과는 항상 활성(active) 상태의 상품만 포함
- 이미 구매한 상품은 추천에서 제외

### 운영 제약
- 모델 학습이 서비스 가용성에 영향을 주지 않아야 한다 (별도 프로세스 또는 배치)
- 모든 추천 API 호출은 로깅하여 성능 지표 계산에 활용
- 모델 버전 관리로 롤백 가능해야 한다
