# 구현 계획 (Phase 1)

## 개발 마일스톤

Phase 1은 4개의 스프린트(각 1주)로 나누어 진행한다.

---

프로젝트 기반 구조를 세우고, 사용자 인증 시스템을 완성한다.

### 태스크

| # | 태스크 | 예상 시간 | 우선순위 |
|---|--------|-----------|----------|

| 1.2 | shadcn/ui 설치 및 테마 설정 | 1h | P0 |
| 1.3 | Prisma 설정 + PostgreSQL 연결 + 스키마 작성 | 2h | P0 |
| 1.4 | NextAuth.js 설정 (Credentials Provider, JWT) | 3h | P0 |
| 1.5 | 회원가입 API + 폼 페이지 | 3h | P0 |
| 1.6 | 로그인/로그아웃 페이지 | 2h | P0 |
| 1.7 | 인증 미들웨어 (보호 라우트) | 1h | P0 |
| 1.8 | 레이아웃 컴포넌트 (Header, BottomNav, MainLayout) | 3h | P0 |
| 1.9 | Vitest + React Testing Library 설정 | 1h | P0 |
| 1.10 | 인증 관련 테스트 작성 | 2h | P0 |

### 완료 기준
- 사용자가 회원가입 후 로그인/로그아웃할 수 있다
- 비인증 사용자는 로그인 페이지로 리다이렉트된다
- 테스트 통과율 100%

## TDD 전략

모든 구현은 TDD 사이클을 준수한다:

### 단위 테스트 (Vitest)
- 서비스 계층 로직 (PostService, FollowService, NotificationService 등)
- Zod 검증 스키마
- 유틸리티 함수

### 통합 테스트 (Vitest + Prisma)
- API Route Handler 테스트 (요청 → 응답 전체 흐름)
- 인증 플로우
- DB 연동 테스트 (테스트용 DB 사용)

### E2E 테스트 (Playwright)
- 회원가입 → 로그인 → 게시물 업로드 → 피드 확인
- 팔로우 → 피드에 게시물 표시
- 좋아요/댓글 → 알림 확인

### 테스트 커버리지 목표
- 전체: 80% 이상
- 서비스 계층: 90% 이상
- API 라우트: 85% 이상
