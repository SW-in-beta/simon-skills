# 요구사항 분석

## 기능 요구사항 (Functional Requirements)

### FR-1: 사용자 인증
| ID | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-1.1 | 이메일/비밀번호로 회원가입할 수 있다 | P0 |
| FR-1.2 | 이메일/비밀번호로 로그인할 수 있다 | P0 |
| FR-1.3 | 로그아웃할 수 있다 | P0 |
| FR-1.4 | 프로필 사진, 이름, 소개글을 설정할 수 있다 | P0 |
| FR-1.5 | 비밀번호를 변경할 수 있다 | P1 |
| FR-1.6 | 사용자명(username)은 고유해야 한다 | P0 |

### FR-2: 게시물 (Post)
| ID | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-2.1 | 사진을 업로드하여 게시물을 작성할 수 있다 | P0 |
| FR-2.2 | 게시물에 캡션(텍스트)을 작성할 수 있다 | P0 |
| FR-2.3 | 자신의 게시물을 삭제할 수 있다 | P0 |
| FR-2.4 | 단일 게시물 상세 페이지를 볼 수 있다 | P0 |
| FR-2.5 | 이미지는 JPEG, PNG, WebP 형식을 지원한다 | P0 |
| FR-2.6 | 이미지 최대 크기는 10MB이다 | P0 |

### FR-3: 피드
| ID | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-3.1 | 로그인 후 팔로우한 사용자의 게시물을 볼 수 있다 | P0 |
| FR-3.2 | 피드는 최신순으로 정렬된다 | P0 |
| FR-3.3 | 무한 스크롤로 추가 게시물을 불러온다 | P0 |
| FR-3.4 | 자신의 게시물도 피드에 표시된다 | P0 |

### FR-4: 팔로우
| ID | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-4.1 | 다른 사용자를 팔로우할 수 있다 | P0 |
| FR-4.2 | 팔로우를 취소(언팔로우)할 수 있다 | P0 |
| FR-4.3 | 팔로워/팔로잉 목록을 볼 수 있다 | P0 |
| FR-4.4 | 팔로워/팔로잉 수를 프로필에서 볼 수 있다 | P0 |

### FR-5: 좋아요
| ID | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-5.1 | 게시물에 좋아요를 누를 수 있다 | P0 |
| FR-5.2 | 좋아요를 취소할 수 있다 | P0 |
| FR-5.3 | 게시물의 좋아요 수를 볼 수 있다 | P0 |
| FR-5.4 | 자신이 좋아요를 눌렀는지 표시된다 | P0 |

### FR-6: 댓글
| ID | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-6.1 | 게시물에 댓글을 작성할 수 있다 | P0 |
| FR-6.2 | 자신의 댓글을 삭제할 수 있다 | P0 |
| FR-6.3 | 게시물의 댓글 목록을 볼 수 있다 | P0 |
| FR-6.4 | 댓글 수를 볼 수 있다 | P0 |

### FR-7: 알림
| ID | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-7.1 | 누군가 나를 팔로우하면 알림을 받는다 | P0 |
| FR-7.2 | 내 게시물에 좋아요가 눌리면 알림을 받는다 | P0 |
| FR-7.3 | 내 게시물에 댓글이 달리면 알림을 받는다 | P0 |
| FR-7.4 | 알림 목록을 볼 수 있다 | P0 |
| FR-7.5 | 읽지 않은 알림 수를 볼 수 있다 | P0 |
| FR-7.6 | 알림을 읽음 처리할 수 있다 | P1 |

### FR-8: 사용자 프로필
| ID | 요구사항 | 우선순위 |
|------|----------|----------|
| FR-8.1 | 사용자 프로필 페이지에서 해당 사용자의 게시물 그리드를 볼 수 있다 | P0 |
| FR-8.2 | 게시물 수, 팔로워 수, 팔로잉 수를 볼 수 있다 | P0 |
| FR-8.3 | 사용자를 검색할 수 있다 (사용자명/이름) | P1 |

## 비기능 요구사항 (Non-Functional Requirements)

### NFR-1: 성능
| ID | 요구사항 | 목표값 |
|------|----------|--------|
| NFR-1.1 | 피드 페이지 초기 로딩 | < 2초 |
| NFR-1.2 | 이미지 업로드 응답 | < 5초 (10MB 기준) |
| NFR-1.3 | API 응답 시간 (일반) | < 500ms |
| NFR-1.4 | 동시 사용자 수 (Phase 1) | 100명 |

### NFR-2: 보안
| ID | 요구사항 |
|------|----------|
| NFR-2.1 | 비밀번호는 bcrypt로 해시 저장한다 |
| NFR-2.2 | JWT 토큰 기반 인증을 사용한다 |
| NFR-2.3 | API는 인증된 사용자만 접근 가능하다 (공개 엔드포인트 제외) |
| NFR-2.4 | 이미지 업로드 시 파일 타입을 검증한다 |
| NFR-2.5 | SQL Injection, XSS를 방지한다 |

### NFR-3: 사용성
| ID | 요구사항 |
|------|----------|
| NFR-3.1 | 반응형 디자인으로 모바일/데스크톱 모두 지원한다 |
| NFR-3.2 | 직관적인 UI로 별도 교육 없이 사용 가능해야 한다 |

### NFR-4: 확장성
| ID | 요구사항 |
|------|----------|
| NFR-4.1 | 향후 기능 추가가 용이한 모듈형 구조로 설계한다 |
| NFR-4.2 | 이미지 스토리지는 클라우드 기반으로 확장 가능해야 한다 |
