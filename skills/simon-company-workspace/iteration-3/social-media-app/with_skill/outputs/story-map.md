## Walking Skeleton (Sprint 1 목표)

핵심 흐름이 끝에서 끝까지 동작하는 최소 버전. 각 Activity의 최소 필수 Task를 수평으로 연결한다.

```
[이메일 가입] → [로그인] → [사진 게시물 작성] → [피드 조회] → [프로필 조회]
   US-001        US-002       US-003              US-004         US-008
```

**Walking Skeleton의 핵심:**
- 사용자가 계정을 만들고 (US-001)
- 로그인하고 (US-002)
- 사진을 올리고 (US-003)
- 피드에서 게시물을 보고 (US-004)
- 프로필에서 게시물 목록을 확인할 수 있다 (US-008)

이 흐름만으로도 "사진 공유 서비스"의 핵심 가치를 전달한다.

## Activity별 상세 흐름

### Activity 1: 가입/인증

```mermaid
graph TD
    A[랜딩 페이지] --> B{계정 있음?}
    B -->|No| C[회원가입 페이지]
    B -->|Yes| D[로그인 페이지]
    C --> E[이메일/비밀번호/사용자명/표시이름 입력]
    E --> F{유효성 검증}
    F -->|실패| E
    F -->|성공| G[계정 생성 → 자동 로그인]
    D --> H[이메일/비밀번호 입력]
    H --> I{인증 확인}
    I -->|실패| H
    I -->|성공| G
    G --> J[메인 피드]
```

### Activity 2: 콘텐츠 공유

```mermaid
graph TD
    A[게시물 작성 버튼] --> B[사진 선택]
    B --> C{파일 검증}
    C -->|실패 - 형식/크기| B
    C -->|성공| D[미리보기 + 캡션 입력]
    D --> E[게시 버튼 클릭]
    E --> F[업로드 진행률 표시]
    F --> G[게시 완료]
    G --> H[피드에 표시]
    G --> I[프로필 그리드에 표시]
```

### Activity 3: 소셜 활동

```mermaid
graph TD
    A[피드의 게시물] --> B[하트 아이콘 클릭]
    B --> C[좋아요 토글]
    C --> D[작성자에게 알림]

    A --> E[댓글 영역]
    E --> F[댓글 입력 + 게시]
    F --> G[댓글 목록에 추가]
    G --> D

    H[프로필 페이지] --> I[팔로우 버튼]
    I --> J[팔로우/언팔로우 토글]
    J --> K[팔로워에게 알림]
    J --> L[피드에 해당 사용자 게시물 포함/제외]
```

### Activity 4: 프로필/설정

```mermaid
graph TD
    A[프로필 페이지] --> B[프로필 정보 표시]
    B --> C[게시물 그리드]
    B --> D[팔로워/팔로잉 수]
    B --> E{본인 프로필?}
    E -->|Yes| F[프로필 편집 버튼]
    E -->|No| G[팔로우 버튼]
    F --> H[편집 페이지]
    H --> I[사진/이름/사용자명/자기소개 수정]
    I --> J[저장]
```

### Activity 5: 발견

```mermaid
graph TD
    A[검색바] --> B[텍스트 입력]
    B --> C[자동완성 결과]
    C --> D[사용자 클릭 → 프로필 이동]

    E[탐색 페이지] --> F[최신 공개 게시물 그리드]
    F --> G[게시물 클릭 → 상세 보기]
```
