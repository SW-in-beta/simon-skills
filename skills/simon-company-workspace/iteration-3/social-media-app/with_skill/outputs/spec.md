# Feature Specification: Picstory - 소셜 미디어 웹앱

## 1. Overview

### Vision
사용자가 사진을 공유하고, 다른 사용자를 팔로우하며, 좋아요와 댓글로 소통하는 인스타그램 스타일의 소셜 미디어 웹앱. 피드는 팔로우한 사용자의 게시물을 최신순으로 표시하며, 소셜 활동에 대한 알림 기능을 제공한다.

### Target Users
- **일반 사용자**: 사진을 올리고 다른 사용자와 소통하고 싶은 개인
- **콘텐츠 소비자**: 팔로우한 사용자의 사진을 탐색하고 반응하는 사용자

### Goals
- 직관적인 사진 공유 경험 제공
- 팔로우 기반 소셜 네트워크 구축
- 실시간에 가까운 알림으로 사용자 참여 유도
- 모바일 우선 반응형 웹 경험

### Non-Goals
- ML 기반 추천 알고리즘 (최신순 피드로 한정)
- 동영상 업로드/스트리밍
- 스토리(24시간 자동 삭제 콘텐츠)
- 다이렉트 메시지(DM)
- 광고 플랫폼
- 해시태그 기반 탐색 (Phase 1 범위 외)

## 2. User Stories

### P1 — Must-Have (MVP)

#### US-001: 이메일 회원가입
**As a** 신규 사용자, **I want to** 이메일과 비밀번호로 계정을 생성할 수 있도록, **so that** 서비스를 이용할 수 있다.

**Acceptance Criteria:**
- **Given** 가입 페이지에 접속한 상태
- **When** 이메일, 비밀번호(8자 이상, 영문+숫자), 사용자명(3-20자, 영문숫자 및 밑줄), 표시 이름을 입력하고 가입 버튼을 클릭
- **Then** 계정이 생성되고 로그인 상태로 메인 피드 페이지로 이동
- **Given** 이미 등록된 이메일로 가입 시도
- **When** 가입 버튼을 클릭
- **Then** "이미 사용 중인 이메일입니다" 에러 메시지 표시
- **Given** 이미 등록된 사용자명으로 가입 시도
- **When** 가입 버튼을 클릭
- **Then** "이미 사용 중인 사용자명입니다" 에러 메시지 표시

**INVEST 검증:** I(독립) O / N(협상가능) O / V(사용자가 서비스 접근) O / E(S사이즈) O / S(작음) O / T(Given/When/Then) O

---

#### US-002: 로그인/로그아웃
**As a** 기존 사용자, **I want to** 이메일과 비밀번호로 로그인/로그아웃할 수 있도록, **so that** 내 계정에 안전하게 접근할 수 있다.

**Acceptance Criteria:**
- **Given** 로그인 페이지에 접속한 상태
- **When** 올바른 이메일과 비밀번호를 입력하고 로그인 버튼 클릭
- **Then** 로그인 성공, 메인 피드 페이지로 이동, JWT 토큰 발급
- **Given** 로그인 상태
- **When** 로그아웃 버튼 클릭
- **Then** 세션 종료, 로그인 페이지로 이동
- **Given** 잘못된 이메일 또는 비밀번호 입력
- **When** 로그인 버튼 클릭
- **Then** "이메일 또는 비밀번호가 올바르지 않습니다" 에러 메시지 표시 (어떤 필드가 잘못되었는지 구분하지 않음)

**INVEST 검증:** I O / N O / V(계정 접근) O / E(S) O / S O / T O

---

#### US-003: 사진 게시물 작성
**As a** 로그인한 사용자, **I want to** 사진과 캡션을 포함한 게시물을 작성할 수 있도록, **so that** 내 사진을 다른 사용자와 공유할 수 있다.

**Acceptance Criteria:**
- **Given** 로그인 상태에서 게시물 작성 페이지에 접속
- **When** 사진 파일(JPG, PNG, WebP, 최대 10MB)을 선택하고 캡션(최대 2000자, 선택사항)을 입력한 후 게시 버튼 클릭
- **Then** 게시물이 생성되고 내 프로필 및 팔로워의 피드에 표시
- **Given** 사진 없이 게시 시도
- **When** 게시 버튼 클릭
- **Then** "사진을 선택해주세요" 에러 메시지 표시
- **Given** 10MB 초과 파일 선택
- **When** 파일 선택
- **Then** "파일 크기는 10MB 이하여야 합니다" 에러 메시지 표시
- **Given** 업로드 중
- **When** 업로드 진행 중
- **Then** 업로드 진행률 표시

**INVEST 검증:** I O / N O / V(사진 공유) O / E(M) O / S O / T O

---

#### US-004: 피드 조회 (최신순)
**As a** 로그인한 사용자, **I want to** 내가 팔로우한 사용자들의 게시물을 최신순으로 볼 수 있도록, **so that** 관심 있는 사용자들의 최신 콘텐츠를 확인할 수 있다.

**Acceptance Criteria:**
- **Given** 로그인 상태에서 메인 피드 페이지에 접속
- **When** 페이지 로드
- **Then** 팔로우한 사용자의 게시물이 최신순으로 표시 (작성자 프로필 사진, 사용자명, 사진, 캡션, 작성 시간, 좋아요 수, 댓글 수 포함)
- **Given** 피드에 게시물이 20개 이상
- **When** 스크롤을 아래로 내림
- **Then** 무한 스크롤로 추가 게시물 로드 (한 번에 10개씩)
- **Given** 팔로우한 사용자가 없거나 게시물이 없는 경우
- **When** 피드 페이지 접속
- **Then** "팔로우할 사용자를 찾아보세요" 안내 메시지와 함께 추천 사용자 목록 표시 (최근 가입 사용자)

**INVEST 검증:** I O / N O / V(콘텐츠 소비) O / E(M) O / S O / T O

---

#### US-005: 사용자 팔로우/언팔로우
**As a** 로그인한 사용자, **I want to** 다른 사용자를 팔로우/언팔로우할 수 있도록, **so that** 관심 있는 사용자의 게시물을 피드에서 볼 수 있다.

**Acceptance Criteria:**
- **Given** 다른 사용자의 프로필 페이지에 접속
- **When** "팔로우" 버튼 클릭
- **Then** 해당 사용자를 팔로우하고 버튼이 "팔로잉"으로 변경, 상대방에게 팔로우 알림 전송
- **Given** 이미 팔로우한 사용자의 프로필 페이지
- **When** "팔로잉" 버튼 클릭
- **Then** 언팔로우 확인 모달 표시, 확인 시 언팔로우 처리
- **Given** 자기 자신의 프로필
- **When** 프로필 페이지 접속
- **Then** 팔로우 버튼이 표시되지 않음

**INVEST 검증:** I O / N O / V(소셜 연결) O / E(S) O / S O / T O

#### US-006: 좋아요
**As a** 로그인한 사용자, **I want to** 게시물에 좋아요를 누르거나 취소할 수 있도록, **so that** 마음에 드는 게시물에 반응할 수 있다.

**Acceptance Criteria:**
- **Given** 피드 또는 게시물 상세에서 좋아요를 누르지 않은 게시물
- **When** 하트 아이콘 클릭
- **Then** 좋아요 추가, 하트가 채워진 상태로 변경, 좋아요 수 +1, 게시물 작성자에게 알림 전송
- **Given** 이미 좋아요를 누른 게시물
- **When** 하트 아이콘 클릭
- **Then** 좋아요 취소, 하트가 빈 상태로 변경, 좋아요 수 -1
- **Given** 게시물의 좋아요 수 표시
- **When** 좋아요 수 영역 클릭
- **Then** 좋아요를 누른 사용자 목록 표시

**INVEST 검증:** I O / N O / V(소셜 반응) O / E(S) O / S O / T O

---

#### US-007: 댓글
**As a** 로그인한 사용자, **I want to** 게시물에 댓글을 달 수 있도록, **so that** 게시물에 대한 의견을 나눌 수 있다.

**Acceptance Criteria:**
- **Given** 게시물 하단의 댓글 영역
- **When** 댓글 텍스트(1-500자)를 입력하고 게시 버튼 클릭
- **Then** 댓글이 추가되고 댓글 목록에 즉시 표시, 게시물 작성자에게 알림 전송
- **Given** 게시물에 댓글이 3개 이상
- **When** 피드에서 해당 게시물 확인
- **Then** 최근 2개 댓글만 표시, "댓글 N개 모두 보기" 링크 표시
- **Given** 내가 작성한 댓글
- **When** 댓글 삭제 버튼 클릭
- **Then** 삭제 확인 후 댓글 제거, 댓글 수 -1

**INVEST 검증:** I O / N O / V(소셜 소통) O / E(M) O / S O / T O

---

#### US-008: 사용자 프로필 조회
**As a** 사용자, **I want to** 다른 사용자의 프로필을 볼 수 있도록, **so that** 그 사용자의 게시물과 정보를 확인할 수 있다.

**Acceptance Criteria:**
- **Given** 사용자명을 클릭하거나 프로필 URL로 접속
- **When** 프로필 페이지 로드
- **Then** 프로필 사진, 표시 이름, 사용자명, 자기소개, 게시물 수, 팔로워 수, 팔로잉 수, 게시물 그리드(썸네일) 표시
- **Given** 프로필의 게시물 그리드
- **When** 게시물 썸네일 클릭
- **Then** 해당 게시물 상세 모달 또는 페이지로 이동
- **Given** 팔로워/팔로잉 수 영역 클릭
- **When** 클릭
- **Then** 해당 목록(팔로워 또는 팔로잉) 표시

**INVEST 검증:** I O / N O / V(사용자 정보 확인) O / E(M) O / S O / T O

---

#### US-009: 프로필 편집
**As a** 로그인한 사용자, **I want to** 내 프로필 정보를 수정할 수 있도록, **so that** 최신 정보를 유지할 수 있다.

**Acceptance Criteria:**
- **Given** 내 프로필 페이지에서 "프로필 편집" 버튼 클릭
- **When** 편집 페이지로 이동
- **Then** 프로필 사진, 표시 이름, 사용자명, 자기소개(최대 150자) 편집 가능
- **Given** 프로필 사진 변경
- **When** 새 사진 파일 선택(JPG, PNG, 최대 5MB)
- **Then** 프로필 사진이 즉시 미리보기로 변경, 저장 시 반영
- **Given** 변경된 정보 저장
- **When** 저장 버튼 클릭
- **Then** 프로필 업데이트 완료, 성공 메시지 표시

**INVEST 검증:** I O / N O / V(프로필 관리) O / E(S) O / S O / T O

---

#### US-010: 알림
**As a** 로그인한 사용자, **I want to** 내 게시물에 대한 좋아요, 댓글, 새 팔로워 알림을 받을 수 있도록, **so that** 소셜 활동을 놓치지 않을 수 있다.

**Acceptance Criteria:**
- **Given** 다른 사용자가 내 게시물에 좋아요/댓글을 달거나 나를 팔로우
- **When** 해당 이벤트 발생
- **Then** 알림이 생성되고 네비게이션 바의 알림 아이콘에 읽지 않은 알림 수 표시 (빨간 뱃지)
- **Given** 알림 아이콘 클릭
- **When** 알림 목록 페이지/드롭다운 표시
- **Then** 알림 목록을 최신순으로 표시 (알림 유형, 관련 사용자, 시간, 관련 게시물 썸네일 포함)
- **Given** 알림 항목 클릭
- **When** 클릭
- **Then** 해당 컨텍스트로 이동 (좋아요/댓글 → 게시물, 팔로우 → 사용자 프로필), 해당 알림 읽음 처리
- **Given** 알림이 없는 경우
- **When** 알림 목록 접속
- **Then** "아직 알림이 없습니다" 메시지 표시

**INVEST 검증:** I O / N O / V(소셜 활동 인지) O / E(M) O / S O / T O

### P2 — Nice-to-Have

#### US-011: 사용자 검색
**As a** 로그인한 사용자, **I want to** 사용자명 또는 표시 이름으로 다른 사용자를 검색할 수 있도록, **so that** 팔로우하고 싶은 사용자를 찾을 수 있다.

**Acceptance Criteria:**
- **Given** 상단 검색바에 텍스트 입력
- **When** 2자 이상 입력
- **Then** 사용자명/표시 이름과 일치하는 사용자를 실시간 자동완성으로 표시 (프로필 사진, 사용자명, 표시 이름)
- **Given** 검색 결과에서 사용자 클릭
- **When** 클릭
- **Then** 해당 사용자 프로필 페이지로 이동

**INVEST 검증:** I O / N O / V(사용자 발견) O / E(S) O / S O / T O

---

#### US-012: 게시물 삭제
**As a** 게시물 작성자, **I want to** 내가 작성한 게시물을 삭제할 수 있도록, **so that** 더 이상 공유하고 싶지 않은 콘텐츠를 제거할 수 있다.

**Acceptance Criteria:**
- **Given** 내가 작성한 게시물의 더보기 메뉴
- **When** "삭제" 옵션 선택
- **Then** 삭제 확인 모달 표시, 확인 시 게시물 및 관련 좋아요/댓글 모두 삭제
- **Given** 다른 사용자의 게시물
- **When** 더보기 메뉴 확인
- **Then** "삭제" 옵션이 표시되지 않음

**INVEST 검증:** I O / N O / V(콘텐츠 관리) O / E(S) O / S O / T O

---

#### US-013: 탐색 페이지
**As a** 로그인한 사용자, **I want to** 팔로우하지 않은 사용자의 인기 게시물을 탐색할 수 있도록, **so that** 새로운 콘텐츠와 사용자를 발견할 수 있다.

**Acceptance Criteria:**
- **Given** 탐색(Explore) 페이지 접속
- **When** 페이지 로드
- **Then** 최근 게시된 공개 게시물을 그리드 형태로 표시 (좋아요 수 기준 가중치 없이 최신순)
- **Given** 그리드의 게시물 클릭
- **When** 클릭
- **Then** 해당 게시물 상세 모달로 이동

**INVEST 검증:** I O / N O / V(콘텐츠 발견) O / E(S) O / S O / T O

### P3 — Future Enhancement

#### US-014: 소셜 로그인 (Google/Apple)
**As a** 신규 사용자, **I want to** Google 또는 Apple 계정으로 간편하게 가입/로그인할 수 있도록, **so that** 별도의 비밀번호를 기억하지 않아도 된다.

#### US-015: 이미지 필터
**As a** 게시물 작성자, **I want to** 업로드 전 사진에 필터를 적용할 수 있도록, **so that** 더 보기 좋은 사진을 공유할 수 있다.

#### US-016: 비밀번호 재설정
**As a** 비밀번호를 잊은 사용자, **I want to** 이메일을 통해 비밀번호를 재설정할 수 있도록, **so that** 계정에 다시 접근할 수 있다.

## 4. Edge Cases

| ID | 시나리오 | 처리 방식 |
|----|---------|----------|
| EC-001 | 동시에 같은 게시물에 좋아요 요청 (더블 클릭) | 백엔드에서 유니크 제약으로 중복 방지, 프론트엔드에서 optimistic update + 디바운스 |
| EC-002 | 사진 업로드 중 네트워크 끊김 | 업로드 실패 안내, 재시도 버튼 제공 |
| EC-003 | 팔로우한 사용자가 계정을 삭제한 경우 | 피드에서 해당 게시물 자동 제외, 팔로잉 수 갱신 |
| EC-004 | 이미 삭제된 게시물에 좋아요/댓글 시도 | "게시물을 찾을 수 없습니다" 에러 표시, 피드에서 해당 게시물 제거 |
| EC-005 | 매우 긴 캡션/댓글의 렌더링 | 피드에서는 3줄까지 표시 후 "더 보기" 링크, 상세에서는 전체 표시 |
| EC-006 | 프로필 사진이 없는 사용자 | 기본 아바타 이미지(이니셜 기반) 표시 |
| EC-007 | 지원하지 않는 이미지 형식 업로드 | "JPG, PNG, WebP 형식만 지원합니다" 에러 메시지 |
| EC-008 | 동일 사용자에게 중복 팔로우 요청 | 백엔드에서 유니크 제약으로 중복 방지, 409 Conflict 응답 |
| EC-009 | 알림이 매우 많은 경우 (수백 개) | 페이지네이션 (30개씩), 30일 이전 알림 자동 정리 |
| EC-010 | 빈 피드 (신규 가입 직후) | "팔로우할 사용자를 찾아보세요" 안내 + 추천 사용자 목록 |

## 5. Key Entities

```
User
├── id (UUID)
├── email (unique)
├── username (unique)
├── display_name
├── bio
├── profile_image_url
├── created_at
└── updated_at

Post
├── id (UUID)
├── user_id (FK → User)
├── image_url
├── thumbnail_url
├── caption
├── like_count (denormalized)
├── comment_count (denormalized)
├── created_at
└── updated_at

Follow
├── id (UUID)
├── follower_id (FK → User)
├── following_id (FK → User)
├── created_at
└── UNIQUE(follower_id, following_id)

Like
├── id (UUID)
├── user_id (FK → User)
├── post_id (FK → Post)
├── created_at
└── UNIQUE(user_id, post_id)

Comment
├── id (UUID)
├── user_id (FK → User)
├── post_id (FK → Post)
├── content
├── created_at
└── updated_at

Notification
├── id (UUID)
├── recipient_id (FK → User)
├── actor_id (FK → User)
├── type (LIKE | COMMENT | FOLLOW)
├── post_id (FK → Post, nullable)
├── is_read (boolean)
├── created_at
└── INDEX(recipient_id, is_read, created_at)
```

**Entity Relationships:**
- User 1:N Post (한 사용자가 여러 게시물 작성)
- User N:M User via Follow (팔로우 관계)
- User N:M Post via Like (좋아요 관계)
- User 1:N Comment, Post 1:N Comment (댓글)
- User 1:N Notification (알림 수신)
