# Discovery Interview Protocol - 인터뷰 산출물

## 메타 정보

- **모드**: 의뢰 모드 (Discovery Interview Protocol)
- **프로토콜**: Phase 1-A' (planning-design.md 참조)
- **인터뷰어**: PM 팀 리드
- **상태**: 인터뷰 질문 준비 완료 (사용자 응답 대기)

---

## Stage 1: 비전 & 맥락 (Vision & Context)

> 사용자의 큰 그림을 이해한다. 기능이 아니라 **문제와 목적**에 집중한다.
> 한 번에 **2개 이하**의 질문만 한다.

### Round 1-1 (첫 번째 질문 세트)

안녕하세요! 고객 데이터를 더 잘 활용하고 싶다는 방향, 잘 이해했습니다. 구체적인 형태를 함께 찾아보겠습니다.

먼저 큰 그림부터 여쭤볼게요:

**Q1.** 회사에서 어떤 종류의 고객 데이터를 쌓고 계신가요? 예를 들어 구매 이력, 행동 로그, 고객 프로필, 상담 기록 등 어떤 것들이 있는지 알려주시면 방향을 잡는 데 도움이 됩니다.

**Q2.** 현재 이 데이터를 보거나 분석할 때 어떤 방식을 쓰고 계신가요? (예: 엑셀로 내려받아서 보고 있다, 특정 BI 도구를 쓰고 있다, 개발자에게 요청해서 쿼리를 돌린다 등)

---

### Round 1-2 (예상 후속 질문 세트)

> 사용자의 답변을 기반으로 비전 요약을 작성하고 확인받은 후 진행할 질문들

**Q3.** "직원들이 데이터를 쉽게 볼 수 있으면 좋겠다"고 하셨는데, 어떤 직원들이 데이터를 봐야 하나요? (예: 마케팅팀, 영업팀, CS팀, 경영진 등) 팀마다 보고 싶은 데이터가 다를 수 있어서요.

**Q4.** "고객한테 더 좋은 경험을 주고 싶다"는 부분이 특히 궁금합니다. 현재 고객이 불편해하거나 아쉬워하는 점이 있나요? 혹은 경쟁사에서 잘 하고 있어서 따라하고 싶은 것이 있나요?

---

### [비전 요약 초안 - 사용자 확인 필요]

> 이 단계에서 PM은 사용자 답변을 종합하여 아래와 같은 형태의 비전 요약을 작성하고 확인받습니다.

```
비전 요약 (초안 - 답변 후 작성 예정):
- 배경: [회사가 쌓고 있는 데이터의 종류와 현재 활용 방식]
- 문제: [현재 데이터 활용의 구체적 어려움]
- 목표: [데이터를 통해 달성하고 싶은 것]
- 대상 사용자: [내부 직원 (어떤 팀?) + 외부 고객]
- 성공의 모습: [한 문장]
```

## Stage 2: 사용자 여정 탐색 (User Journey Discovery)

> 핵심 사용자의 관점에서 서비스 이용 흐름을 함께 그린다.
> Stage 1 답변 확인 후 진행

### Round 2-1

**Q5.** 직원이 이 시스템에 로그인하면 가장 먼저 보고 싶은 화면은 어떤 건가요? 예를 들어, 오늘의 핵심 지표 요약? 특정 고객의 상세 정보? 전체 트렌드 그래프?

**Q6.** 고객 입장에서는 어떤가요? 고객이 데이터를 직접 보게 되나요, 아니면 데이터를 기반으로 추천이나 알림 같은 것을 받게 되나요?

### Round 2-2

**Q7.** 직원이 데이터를 볼 때, "보기만" 하면 되나요? 아니면 데이터를 기반으로 어떤 액션을 취해야 하나요? (예: 특정 조건의 고객에게 쿠폰 발송, 이탈 위험 고객 표시 후 연락 등)

**Q8.** 여러 유형의 사용자가 있다면, 각각의 핵심 시나리오를 하나씩만 말씀해 주실 수 있나요? (예: "마케터는 이번 달 캠페인 성과를 보고 다음 캠페인을 기획한다")

---

### [사용자 여정 초안 - 답변 후 작성 예정]

```
[내부 직원 여정]
[로그인] --> [대시보드 확인] --> [상세 데이터 탐색] --> [액션 실행?] --> [결과 확인]

[고객 여정]
[서비스 이용] --> [개인화된 경험] --> [???] --> [만족/재방문]
```

## Stage 3: 기능별 심층 인터뷰 (Feature Deep-Dive)

> Stage 2에서 파악한 사용자 여정의 각 단계를 하나씩 깊게 파고든다.
> Stage 2 답변 확인 후 진행

### Round 3-1 (직원용 기능 탐색)

**Q9.** [대시보드 관련] 대시보드에 표시할 핵심 지표(KPI)가 있나요? 현재 경영진이나 팀장이 주기적으로 확인하는 수치가 있다면 알려주세요.

**Q10.** [데이터 탐색 관련] 직원이 데이터를 검색하거나 필터링할 때, 어떤 기준으로 찾나요? (예: 기간별, 고객 세그먼트별, 상품별, 지역별 등)

### Round 3-2 (고객용 기능 탐색)

**Q11.** [고객 경험 관련] 고객 데이터를 기반으로 고객에게 제공하고 싶은 것이 구체적으로 어떤 건가요? 몇 가지 예를 들어보겠습니다. 가장 가까운 것을 골라주세요:
  - A) 개인화된 상품/콘텐츠 추천
  - B) 고객 자신의 활동 요약/리포트 (예: 넷플릭스의 "올해 시청한 콘텐츠")
  - C) 맞춤형 알림/쿠폰/프로모션
  - D) 고객 등급/리워드 시스템
  - E) 기타 (설명해 주세요)

**Q12.** 이 기능이 기존 서비스(앱/웹)에 추가되는 건가요, 아니면 새로운 독립 서비스로 만드는 건가요?

### Round 3-3 (추가 기능 탐색)

**Q13.** 데이터 보안과 접근 권한이 중요할 것 같은데, 직원 간에도 볼 수 있는 데이터가 다른가요? (예: CS팀은 개별 고객 데이터만, 경영진은 전체 통계만)

**Q14.** 실시간 데이터가 필요한가요, 아니면 하루에 한 번 업데이트되는 정도면 충분한가요?

### [User Story 초안 - 답변 후 작성 예정]

> 각 기능 인터뷰 후, 해당 기능의 User Story 초안을 즉석에서 보여주고 확인받는다.

```
예시 (답변에 따라 실제 작성):

US-001: 대시보드 핵심 지표 확인
As a 마케팅 팀장, I want to 로그인 후 오늘의 핵심 KPI를 한눈에 볼 수 있도록,
so that 별도로 데이터를 요청하지 않고도 현재 상황을 빠르게 파악할 수 있다.
- Given 마케팅 팀장이 대시보드에 접속했을 때
- When 메인 페이지가 로드되면
- Then 매출, DAU, 전환율 등 핵심 지표가 카드 형태로 표시된다
- Edge case: 데이터 수집 장애 시 -> "마지막 업데이트: X시간 전" 경고 표시

확인: 이 이해가 맞나요? 맞다면 다음 기능으로 넘어가겠습니다.
```

## Stage 4: 우선순위 & 범위 조정 (Prioritization & Scoping)

> 모든 기능이 나왔으면 함께 우선순위를 정한다.
> Stage 3 완료 후 진행

### Round 4-1

```
PM:
"지금까지 나온 기능을 정리하면 이렇습니다:

[인터뷰 결과 기반 기능 목록 - 답변 후 작성]

예시:
1. 직원용 KPI 대시보드 - 핵심 지표를 실시간으로 확인
2. 데이터 탐색/필터링 - 다양한 기준으로 고객 데이터 검색
3. 고객 세그먼트 분석 - 고객 그룹별 행동 패턴 비교
4. 리포트 자동 생성 - 주간/월간 리포트 자동 생성 및 공유
5. 고객 맞춤 추천 - 고객 행동 기반 개인화된 추천
6. 고객 활동 요약 - 고객에게 자신의 활동 리포트 제공
7. 역할 기반 접근 권한 - 직원 역할에 따른 데이터 접근 제한
8. 알림/경보 시스템 - 특이 데이터 패턴 발생 시 자동 알림

이 중에서:
- 반드시 있어야 하는 것 (P1 - MVP)은?
- 있으면 좋은 것 (P2)은?
- 나중에 추가해도 되는 것 (P3)은?

또한, 빠진 기능이 있을까요?"
```

---

## Stage 5: 제약 조건 & 품질 기준 (Constraints & Quality)

> 비기능 요구사항과 프로젝트 제약을 확인한다.
> Stage 4 완료 후 진행

### Round 5-1

**Q15.** 이 서비스를 사용하는 직원 수는 어느 정도인가요? 그리고 고객 수는요?

**Q16.** 기존에 사용하고 있는 시스템이 있나요? (예: 고객 데이터가 저장된 DB, CRM 시스템, 분석 도구 등) 새로 만드는 것과 연동이 필요한 시스템이 있으면 알려주세요.

### Round 5-2

**Q17.** 일정이나 마일스톤에 대한 기대가 있나요? (예: 3개월 안에 MVP, 연말까지 전체 완성 등)

**Q18.** 보안 관련해서 특별히 신경 써야 할 부분이 있나요? (예: 개인정보보호법 준수, 특정 인증 필요, 내부 보안 정책 등)

## 인터뷰 원칙 준수 체크리스트

| 원칙 | 준수 여부 | 비고 |
|------|----------|------|
| 한 번에 질문 최대 2개 | O | 모든 라운드에서 2개 이하 유지 |
| 답변을 즉시 구조화하여 되돌려준다 | O | 비전 요약, 사용자 여정, User Story 초안 즉시 제시 |
| "모르겠다"에 선택지 제시 | O | Q11에서 A/B/C/D/E 선택지 제공 |
| 가정을 숨기지 않는다 | O | 비전 요약에서 가정을 명시하고 확인받는 구조 |
| 누적 진행률 표시 | 예정 | Stage 3 이후부터 "현재 X/Y 기능 구체화됨" 표시 예정 |

## 인터뷰 진행 흐름 요약

```
Stage 1: 비전 & 맥락 (Q1-Q4, 2라운드)
    --> 비전 요약 작성 & 확인
Stage 2: 사용자 여정 탐색 (Q5-Q8, 2라운드)
    --> 사용자 여정 초안 작성 & 확인
Stage 3: 기능별 심층 인터뷰 (Q9-Q14, 3라운드)
    --> 각 기능별 User Story 초안 작성 & 확인
Stage 4: 우선순위 & 범위 조정 (1라운드)
    --> P1/P2/P3 분류 & 확인
Stage 5: 제약 조건 & 품질 기준 (Q15-Q18, 2라운드)
    --> 비기능 요구사항 정리

총 예상: 10라운드, 18개 질문
```

## 의뢰 모드 완료 조건 (아직 미충족)

- [ ] 모든 P1 기능에 대해 User Story + Acceptance Criteria가 확인됨
- [ ] 사용자가 "이 범위로 진행해도 됩니다"라고 명시적으로 승인
- [ ] 이후 1-B(Feature Specification)로 진행하여 정식 spec.md 작성
