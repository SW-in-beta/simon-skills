# Constitution: RecSys - 사용자 행동 기반 상품 추천 시스템

## Core Principles

1. **추천 품질 최우선** — 정확하지 않은 추천은 안 하느니만 못하다. Cold Start이든 데이터 부족이든, 항상 합리적인 Fallback을 제공하여 사용자 경험을 해치지 않는다.
2. **API 응답 속도는 타협 불가** — 추천 API는 p95 200ms 이내 응답해야 한다. 모델 추론이 느리면 캐싱이나 사전 계산으로 해결하지, 응답 시간을 포기하지 않는다.
3. **운영 가시성 확보** — 모니터링 없는 ML 시스템은 블랙박스다. 모든 핵심 지표가 대시보드에서 확인 가능해야 하며, 이상 상황은 즉시 감지되어야 한다.
4. **데이터 무결성** — 사용자 행동 데이터는 추천의 원천이다. 데이터 유실이나 오염은 추천 품질에 직접적인 영향을 미치므로 수집-저장-전처리 전 과정에서 무결성을 보장한다.
5. **점진적 가치 전달** — Walking Skeleton부터 시작하여, 각 Sprint마다 동작하는 소프트웨어 증분을 제공한다.

## Quality Gates

- 테스트 커버리지: 80% 이상
- TRP 3라운드 통과 필수
- 추천 API 응답 시간: p95 < 200ms
- 행동 이벤트 처리량: 1,000 events/sec 이상
- 모델 학습 안정성: 학습 성공률 > 95%
- Precision@10 > 0.1 (최소 품질 기준)
- OWASP Top 10 준수 (API 인증, 입력 검증, Rate Limiting)
- Cold Start 사용자에 대한 Fallback 응답 100%

## Constraints

### 기술적 제약
- Collaborative Filtering 기반 (ALS Matrix Factorization) — 사용자 요구사항으로 확정
- REST API 기반 서빙 — 사용자 요구사항으로 확정
- 배치 학습 (일 1회) — 실시간 학습은 P3에서 검토
- 모델 학습과 서빙의 분리 — 학습 중에도 기존 모델로 서빙 지속

### 비즈니스 제약
- 관리자 대시보드는 웹 기반으로 제공
- API 인증은 API Key 기반 (MVP 단계)
- 대시보드 접근은 관리자 계정 인증 필요

### 데이터 제약
- 최소 학습 데이터: 100명 사용자, 50개 상품, 1,000건 상호작용
- 행동 데이터 최소 1년 보존
- 개인정보 직접 저장 금지 (사용자 ID는 외부 시스템의 식별자)

## Architecture Principles

- **모델-서빙 분리**: 학습 파이프라인과 서빙 레이어를 분리하여 학습 실패가 서빙에 영향을 주지 않도록 한다
- **캐싱 전략**: 추천 결과는 적절한 TTL로 캐싱하여 응답 시간을 보장한다
- **Graceful Degradation**: 모델 장애 시 인기 상품 Fallback으로 서비스 연속성을 유지한다
- **이벤트 소싱**: 행동 이벤트는 추가 전용(append-only) 패턴으로 저장하여 데이터 무결성을 보장한다

## Definition of Done (DoD)

모든 Feature는 다음 기준을 모두 충족해야 "완료"로 인정된다:

- [ ] 코드가 lint/format 검사를 통과한다
- [ ] 단위 테스트가 작성되었고 모두 통과한다 (커버리지 80%+)
- [ ] API Contract을 준수한다
- [ ] 에러 핸들링이 구현되었다
- [ ] 로깅이 적절히 추가되었다
- [ ] TDD 사이클(RED-GREEN-REFACTOR)을 따랐다
- [ ] 코드 리뷰(TRP R2)를 통과했다

## Risk Register

| 리스크 | 영향 | 완화 전략 |
|--------|------|----------|
| Cold Start 문제 | 신규 사용자 추천 불가 | 인기 상품 Fallback |
| 데이터 희소성 | 모델 정확도 저하 | 최소 데이터 기준 설정, 암묵적 피드백 활용 |
| 모델 학습 실패 | 추천 품질 정체 | 이전 모델 유지, 학습 실패 알림 |
| API 응답 지연 | 사용자 경험 저하 | 추천 결과 사전 계산/캐싱 |
| 데이터 편향 | 인기 상품 편중 | 탐색-활용 균형 (P2에서 검토) |
