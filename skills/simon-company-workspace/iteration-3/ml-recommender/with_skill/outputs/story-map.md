## Walking Skeleton (Sprint 1 목표)

핵심 흐름이 끝에서 끝까지 동작하는 최소 버전. 각 Activity의 최소 필수 Task를 수평으로 연결한다.

```
[상품 등록] → [사용자 등록] → [행동 이벤트 수집] → [모델 학습] → [추천 요청] → [결과 반환]
 US-004       US-005          US-001              US-003        US-002        US-002
```

**Walking Skeleton 범위:**
1. 상품을 등록할 수 있다 (US-004: 최소 CRUD)
2. 사용자를 등록할 수 있다 (US-005: 등록/조회)
3. 행동 이벤트를 기록할 수 있다 (US-001: 단건 이벤트 수집)
4. 행동 데이터로 CF 모델을 학습할 수 있다 (US-003: ALS 기반 학습)
5. 개인화 추천 결과를 API로 받을 수 있다 (US-002: 추천 서빙 + Cold Start Fallback)

## Release 계획

### Release 1: Walking Skeleton (Sprint 1-2)
핵심 추천 파이프라인이 동작하는 최소 버전.

| Sprint | Features | 팀 |
|--------|----------|-----|
| Sprint 1 | US-004 상품 CRUD, US-005 사용자 관리, US-001 행동 이벤트 수집 (단건), DB 스키마 초기 구축 | Backend, DBA |
| Sprint 2 | US-003 CF 모델 학습, US-002 추천 API (+ Cold Start Fallback), US-001 배치 이벤트 수집 | Backend, ML |

**검증 기준:** 상품 등록 → 사용자 등록 → 행동 기록 → 모델 학습 → 추천 요청까지 전체 파이프라인이 동작한다.

### Release 2: 운영 가시성 (Sprint 3-4)
대시보드와 모델 관리 기능을 추가하여 운영 가능한 시스템을 완성한다.

| Sprint | Features | 팀 |
|--------|----------|-----|
| Sprint 3 | US-006 성능 지표 대시보드 (모델 메트릭 + 서비스 메트릭), 인프라 셋업 | Frontend, Design, DevOps |
| Sprint 4 | US-006 비즈니스 메트릭, US-007 모델 관리 UI, 모니터링/알림 셋업 | Frontend, Backend, DevOps |

**검증 기준:** 대시보드에서 실시간 성능 지표를 확인하고, 모델 학습을 수동으로 트리거할 수 있다.

### Release 3: 고도화 (Sprint 5-6)
추천 품질 개선 및 고급 기능.

| Sprint | Features | 팀 |
|--------|----------|-----|
| Sprint 5 | US-008 유사 상품 추천, US-009 피드백 수집 | Backend, ML |

## Feature 의존성 맵

```
US-004 (상품 관리) ←── US-001 (행동 수집) ←── US-003 (모델 학습) ←── US-002 (추천 API)
US-005 (사용자 관리) ←┘                                              │
                                                                      ├── US-006 (대시보드)
                                                                      ├── US-007 (모델 관리)
                                                                      ├── US-008 (유사 상품)
                                                                      └── US-009 (피드백)
                                                                           └── US-010 (A/B테스트)
```

**핵심 의존성:**
- US-001 → US-004, US-005에 의존 (상품/사용자가 존재해야 이벤트 수집 가능)
- US-003 → US-001에 의존 (행동 데이터가 있어야 학습 가능)
- US-002 → US-003에 의존 (모델이 있어야 추천 가능, 단 Cold Start Fallback은 독립)
- US-006, US-007 → US-003에 의존 (모델이 있어야 지표/관리 의미 있음)
- US-008 → US-003에 의존 (모델의 Item Factor로 유사도 계산)
- US-009 → US-002에 의존 (추천 결과가 있어야 피드백 가능)
