# 구현 계획 (Phase 1)

## 1. Phase 1 범위 요약

Phase 1은 추천 시스템의 핵심 기능을 구현하여 MVP를 완성하는 단계이다.

**포함:**
- 데이터 모델 및 DB 스키마 생성
- 사용자 행동 이벤트 수집 API
- Collaborative Filtering 모델 (User-based, Item-based, ALS)
- 배치 학습 파이프라인
- 추천 서빙 API
- 관리자 대시보드 (기본 지표 모니터링)
- 콜드 스타트 처리 (인기도 기반)

**제외 (Phase 2 이후):**
- 실시간 스트리밍 파이프라인
- A/B 테스트 프레임워크
- Content-based Filtering
- 딥러닝 모델
- MLOps 자동화

## 4. 리스크 및 대응

| 리스크 | 영향 | 확률 | 대응 |
|--------|------|------|------|
| 행동 데이터 부족 (콜드 스타트) | 높음 | 중간 | 인기도 기반 폴백, 시드 데이터 활용 |
| 추천 응답 시간 초과 | 높음 | 낮음 | 사전 계산 + 캐싱, 모델 최적화 |
| User-Item Matrix 메모리 초과 | 중간 | 낮음 | Sparse Matrix, 데이터 필터링 강화 |
| 모델 성능 미달 | 높음 | 중간 | 하이퍼파라미터 튜닝, 앙상블 가중치 조정 |
| 대시보드 개발 지연 | 낮음 | 중간 | 핵심 지표만 우선 구현, 상세 차트 후순위 |

## 5. 성공 기준 (Phase 1 완료 판단)

- [ ] 이벤트 수집 API가 동작하고 데이터가 DB에 저장됨
- [ ] CF 모델 3종이 학습되고 오프라인 평가 지표가 기준 이상
- [ ] 배치 학습 파이프라인이 스케줄에 따라 자동 실행
- [ ] 추천 API가 p95 < 200ms 응답 시간으로 동작
- [ ] 콜드 스타트 사용자에게 인기도 기반 추천이 제공
- [ ] 관리자 대시보드에서 CTR, 전환율, 커버리지 등 핵심 지표 확인 가능
- [ ] 모델 목록 및 성능 비교가 대시보드에서 가능
- [ ] 수동 재학습 트리거가 대시보드에서 가능
- [ ] 테스트 커버리지 80% 이상
- [ ] E2E 통합 테스트 통과
