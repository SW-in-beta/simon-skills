# Scope Guard 평가 결과

## 사용자 요청

"CSV 파일을 읽어서 JSON으로 변환하는 CLI 도구를 만들어줘. 인코딩 자동 감지, 컬럼 필터링, 대용량 파일 스트리밍 지원이 필요해."

## 프로젝트 유형 판별 (Phase 0-A)

- **유형**: Greenfield (새 프로젝트)
- **성격**: CLI 데이터 처리 도구

## Scope Guard 평가 (4가지 기준)

| 기준 | simon-company 조건 | 이 프로젝트 | 판정 |
|------|-------------------|------------|------|
| 필요 팀 수 | 3개 이상 (예: BE+FE+DevOps) | **2개** (Backend + QA) | 리다이렉트 |
| 인프라 필요 | CI/CD, 배포, 모니터링 필요 | **불필요** (로컬 실행 CLI) | 리다이렉트 |
| 디자인 필요 | UI/UX 설계 필요 | **불필요** (CLI, UI 없음) | 리다이렉트 |
| 예상 Feature 수 | 5개 이상 | **3-4개** | 리다이렉트 |

## 상세 분석

### 필요 팀: 2개

1. **Backend (필수)**: CLI 도구의 핵심 로직 구현
   - CSV 파싱, 인코딩 감지, 컬럼 필터링, 스트리밍, JSON 출력
2. **QA (필수)**: 테스트 작성 및 검증
   - 다양한 인코딩 파일 테스트, 대용량 파일 성능 테스트, 엣지 케이스

**불필요한 팀들:**
- PM: 요구사항이 이미 명확 (CLI 도구, 기능 3개 명시)
- CTO: 기술 의사결정 단순 (단일 언어 CLI 프로젝트)
- Design: UI 없음
- Frontend: UI 없음
- DBA: DB 없음
- DevOps: 배포/인프라 불필요
- ML: ML 기능 없음

### 예상 Feature 목록: 3-4개

1. **CSV 파싱 및 JSON 변환** (핵심 기능)
2. **인코딩 자동 감지** (chardet/charset-normalizer 활용)
3. **컬럼 필터링** (특정 컬럼만 선택/제외)
4. **대용량 파일 스트리밍** (메모리 효율적 처리)

### 인프라 요구사항: 없음

- 로컬에서 실행하는 독립 CLI 도구
- 배포 파이프라인, 컨테이너화, 모니터링 불필요
- 패키지 배포(pip/npm)는 선택적이며 simon-company 수준의 인프라가 아님

## 결론

**판정: 리다이렉트 (simon-company 부적합)**

4가지 기준 모두에서 simon-company의 적용 범위를 벗어남. 이 프로젝트에 simon-company를 적용하면 불필요한 팀 편성, 과도한 프로세스(8단계 파이프라인, TRP 3라운드 검토, 팀간 계약 등), 토큰 낭비가 발생함.

## 리다이렉트 제안

### 1순위: simon-bot-pm

- **이유**: Feature가 3-4개로, 단순 단일 작업은 아니지만 다중 팀 협업이 필요하지 않음
- **적합성**: simon-bot-pm은 1-2개 팀 규모의 프로젝트에 적합하며, PM 관점의 기획 + 구현을 하나의 흐름으로 처리 가능
- **이점**: 적절한 수준의 기획(spec) + TDD 기반 구현 + 테스트 검증

### 2순위: simon-bot

- **이유**: 기능이 상대적으로 간단하고 명확하다면, simon-bot만으로도 충분히 구현 가능
- **적합성**: 요구사항이 이미 명확하게 정의되어 있어 추가 기획이 최소화됨
