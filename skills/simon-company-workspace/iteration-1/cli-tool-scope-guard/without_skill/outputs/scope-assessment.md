# 프로젝트 범위 평가: CSV-to-JSON CLI 도구

## 요청 요약

CSV 파일을 JSON으로 변환하는 CLI 도구. 핵심 요구사항 3가지:
1. 인코딩 자동 감지
2. 컬럼 필터링
3. 대용량 파일 스트리밍 지원

## 범위 분류: **소규모 단일 개발자 프로젝트**

이 프로젝트는 1명의 개발자가 1~3일 내에 완료할 수 있는 소규모 유틸리티 도구이다.

## 근거

### 기술적 복잡도: 낮음~중간

| 기능 | 복잡도 | 설명 |
|------|--------|------|
| CSV 파싱 | 낮음 | 표준 라이브러리(Go: `encoding/csv`, Python: `csv`) 또는 성숙한 서드파티 라이브러리 존재 |
| JSON 출력 | 낮음 | 표준 라이브러리로 충분 |
| 인코딩 자동 감지 | 중간 | Go: `golang.org/x/text/encoding`, Python: `chardet`/`charset-normalizer` 등 기존 라이브러리 활용 |
| 컬럼 필터링 | 낮음 | CLI 플래그로 컬럼명/인덱스 지정, 단순 필터링 로직 |
| 대용량 스트리밍 | 중간 | 행 단위 스트리밍 처리, 메모리 효율적 JSON 출력(NDJSON 또는 JSON array streaming) |
| CLI 인터페이스 | 낮음 | Go: `cobra`/`flag`, Python: `click`/`argparse` |

### 외부 의존성: 없음

- 네트워크 통신 없음
- 데이터베이스 없음
- 외부 API 연동 없음
- 인증/인가 없음

### 비기능 요구사항

- 테스트: 단위 테스트 + 통합 테스트 (다양한 인코딩, 대용량 파일, 엣지 케이스)
- 에러 처리: 잘못된 CSV, 지원하지 않는 인코딩, 존재하지 않는 컬럼명 등
- 성능: 대용량 파일(수 GB)에서도 일정한 메모리 사용량 유지

## 추정 작업량

| 단계 | 예상 시간 |
|------|-----------|
| 프로젝트 초기화 + CLI 프레임워크 | 2시간 |
| CSV 파싱 + JSON 변환 (기본) | 2시간 |
| 인코딩 자동 감지 | 3시간 |
| 컬럼 필터링 | 2시간 |
| 대용량 스트리밍 | 3시간 |
| 테스트 작성 (TDD) | 4시간 |
| 에러 처리 + 엣지 케이스 | 2시간 |
| **합계** | **약 18시간 (2~3일)** |

## 위험 요소

1. **인코딩 감지 정확도**: 자동 감지 라이브러리의 정확도가 100%가 아님. 감지 실패 시 fallback 인코딩 지정 옵션 필요.
2. **CSV 방언(dialect) 차이**: 구분자, 따옴표 처리, 줄바꿈 등 다양한 CSV 형식 대응 필요.
3. **대용량 JSON 출력 형식**: 전체를 하나의 JSON array로 출력하면 스트리밍의 이점이 줄어듦. NDJSON(줄 단위 JSON) 지원 고려.
