# 소셜 미디어 웹앱 기획 산출물 요약

## 프로젝트 개요

인스타그램 스타일의 소셜 미디어 웹앱을 기획하였습니다. 사진 업로드, 팔로우, 좋아요/댓글, 최신순 피드, 알림 기능을 포함하며, Next.js + React + Tailwind CSS 기반의 웹 애플리케이션입니다.

---

## 생성된 산출물 목록

### 1. 팀 구성 및 역할 정의
- **파일:** `01-team-composition.md`
- **내용:** 8명 규모의 팀 구성안. PM 1명, 디자이너 1명, 프론트엔드 2명, 백엔드 2명, DevOps 1명(파트타임), QA 1명(파트타임). 기술 스택 정의 포함.

### 2. 기능 명세서 (Feature Specification)
- **파일:** `02-feature-specification.md`
- **내용:** 8개 기능 도메인, 20개 이상의 사용자 스토리 정의.
  - 인증 및 계정 관리 (회원가입, 로그인, 소셜 로그인, 프로필 편집)
  - 사진 게시 (업로드, 캐러셀, 수정/삭제)
  - 피드 (홈 피드 최신순, 탐색 페이지)
  - 팔로우 시스템 (팔로우/언팔로우, 비공개 계정 요청)
  - 좋아요 (토글, 목록)
  - 댓글 (작성, 삭제, 목록)
  - 알림 (5가지 유형, 실시간 WebSocket)
  - 프로필 페이지
  - 비기능 요구사항 (성능, 보안, 확장성, 접근성, 데이터)

### 3. 프로젝트 계획
- **파일:** `03-project-plan.md`
- **내용:** 12주(6스프린트) 프로젝트 계획.
  - Phase 1 (Sprint 1~2): 기반 구축 - 인증, 게시물 업로드
  - Phase 2 (Sprint 3~4): 핵심 소셜 기능 - 피드, 팔로우, 좋아요, 댓글
  - Phase 3 (Sprint 5~6): 알림 시스템, 폴리시, 런칭 준비
  - 스프린트별 태스크 분배 및 공수 산정 포함
  - 리스크 관리 계획
  - 데이터 모델(ERD) 개요 - 9개 테이블
  - API 엔드포인트 설계 - 27개 엔드포인트

## 핵심 설계 결정

| 항목 | 결정 | 근거 |
|------|------|------|
| 피드 알고리즘 | 최신순 (역시간순) | 요구사항 명시, 구현 단순성 |
| 페이지네이션 | 커서 기반 | 실시간 피드에서 offset 방식의 중복/누락 문제 방지 |
| 이미지 저장 | S3 + CDN | 확장성, 비용 효율, 글로벌 배포 |
| 실시간 알림 | WebSocket | 양방향 통신, 낮은 지연 |
| 삭제 전략 | 소프트 삭제 | 데이터 복구 가능성, 규정 준수 |
| 프론트엔드 | Next.js + shadcn/ui | SSR/SEO 지원, 일관된 디자인 시스템 |
