# 기능 명세서 (Feature Specification)

## 1. 사용자 인증 및 계정 관리

### US-1.1: 회원가입
- **사용자로서** 이메일과 비밀번호로 계정을 생성할 수 있다.
- **인수 조건:**
  - 이메일 형식 검증
  - 비밀번호 최소 8자, 영문+숫자+특수문자 조합
  - 이메일 중복 확인
  - 이메일 인증 링크 발송
  - 사용자명(username) 고유성 검증 (영문, 숫자, 밑줄만 허용)

### US-1.2: 로그인 / 로그아웃
- **사용자로서** 이메일과 비밀번호로 로그인할 수 있다.
- **인수 조건:**
  - 로그인 성공 시 JWT 토큰 발급
  - 로그인 실패 시 에러 메시지 표시
  - 로그아웃 시 토큰 무효화
  - "로그인 유지" 옵션 (Refresh Token)

### US-1.3: 소셜 로그인
- **사용자로서** Google / Apple 계정으로 간편 로그인할 수 있다.
- **인수 조건:**
  - OAuth 2.0 플로우
  - 최초 소셜 로그인 시 사용자명 설정 화면 표시

### US-1.4: 프로필 편집
- **사용자로서** 내 프로필 정보를 수정할 수 있다.
- **인수 조건:**
  - 프로필 사진 업로드/변경 (최대 5MB, jpg/png/webp)
  - 이름, 자기소개(bio, 최대 150자), 웹사이트 URL 편집
  - 계정 공개/비공개 설정

---

## 2. 사진 게시 (Post)

### US-2.1: 사진 업로드
- **사용자로서** 사진을 선택하여 게시물을 작성할 수 있다.
- **인수 조건:**
  - 사진 1~10장 동시 업로드 (캐러셀)
  - 지원 형식: jpg, png, webp, heic
  - 최대 파일 크기: 장당 10MB
  - 업로드 진행률 표시
  - 이미지 리사이징 및 압축 (서버 측)

### US-2.2: 게시물 작성
- **사용자로서** 사진에 설명(캡션)을 추가하여 게시할 수 있다.
- **인수 조건:**
  - 캡션 최대 2,200자
  - 해시태그(#) 지원 및 자동 링크
  - 사용자 멘션(@) 지원 및 자동완성
  - 위치 정보 추가 (선택)
  - 게시 전 미리보기

### US-2.3: 게시물 수정/삭제
- **사용자로서** 내가 올린 게시물을 수정하거나 삭제할 수 있다.
- **인수 조건:**
  - 캡션 수정 가능 (사진 변경 불가)
  - 삭제 시 확인 다이얼로그
  - 삭제 시 관련 좋아요/댓글도 함께 삭제 (소프트 삭제)

---

## 3. 피드 (Feed)

### US-3.1: 홈 피드
- **사용자로서** 내가 팔로우하는 사람들의 최신 게시물을 볼 수 있다.
- **인수 조건:**
  - 팔로잉 사용자의 게시물을 최신순(역시간순)으로 표시
  - 무한 스크롤 (커서 기반 페이지네이션, 한 번에 20개)
  - 게시물 카드: 사용자 프로필 사진, 사용자명, 게시 시간, 사진, 좋아요 수, 댓글 수, 캡션 미리보기
  - 새 게시물 알림 ("새 게시물 보기" 버튼)
  - 팔로잉이 없을 때 추천 계정 표시

### US-3.2: 탐색 페이지 (Explore)
- **사용자로서** 새로운 콘텐츠와 사용자를 발견할 수 있다.
- **인수 조건:**
  - 전체 공개 게시물 중 최신순 그리드 표시
  - 사용자 검색 (사용자명, 이름)
  - 해시태그 검색
  - 검색 결과 페이지네이션

---

## 4. 팔로우 시스템

### US-4.1: 팔로우/언팔로우
- **사용자로서** 다른 사용자를 팔로우하거나 언팔로우할 수 있다.
- **인수 조건:**
  - 팔로우 버튼 클릭으로 즉시 팔로우
  - 비공개 계정은 팔로우 요청 → 승인/거절
  - 언팔로우 시 확인 없이 즉시 처리
  - 자기 자신 팔로우 불가

### US-4.2: 팔로워/팔로잉 목록
- **사용자로서** 내/타인의 팔로워와 팔로잉 목록을 볼 수 있다.
- **인수 조건:**
  - 팔로워/팔로잉 수 표시
  - 목록에서 바로 팔로우/언팔로우 가능
  - 무한 스크롤

### US-4.3: 팔로우 요청 관리
- **비공개 계정 사용자로서** 받은 팔로우 요청을 승인하거나 거절할 수 있다.
- **인수 조건:**
  - 대기 중인 요청 목록 표시
  - 승인/거절 버튼
  - 승인 시 팔로워에 추가 + 알림 발송

---

## 5. 좋아요 (Like)

### US-5.1: 게시물 좋아요/취소
- **사용자로서** 게시물에 좋아요를 누르거나 취소할 수 있다.
- **인수 조건:**
  - 하트 아이콘 클릭 또는 사진 더블클릭으로 좋아요
  - 좋아요 상태 즉시 반영 (Optimistic UI)
  - 좋아요 수 표시
  - 좋아요 취소 가능

### US-5.2: 좋아요 누른 사람 목록
- **사용자로서** 게시물에 좋아요를 누른 사람들을 확인할 수 있다.
- **인수 조건:**
  - 사용자 목록 표시 (프로필 사진, 사용자명, 이름)
  - 목록에서 바로 팔로우 가능

---

## 6. 댓글 (Comment)

### US-6.1: 댓글 작성
- **사용자로서** 게시물에 댓글을 달 수 있다.
- **인수 조건:**
  - 텍스트 댓글 (최대 1,000자)
  - 멘션(@) 지원
  - 댓글 입력 즉시 표시 (Optimistic UI)

### US-6.2: 댓글 삭제
- **사용자로서** 내가 작성한 댓글을 삭제할 수 있다.
- **게시물 작성자로서** 내 게시물의 댓글을 삭제할 수 있다.
- **인수 조건:**
  - 삭제 확인 다이얼로그
  - 소프트 삭제

### US-6.3: 댓글 목록 조회
- **사용자로서** 게시물의 댓글을 최신순으로 볼 수 있다.
- **인수 조건:**
  - 게시물 상세에서 전체 댓글 표시
  - 피드에서는 최신 댓글 2개 미리보기
  - 무한 스크롤

---

## 7. 알림 (Notification)

### US-7.1: 알림 수신
- **사용자로서** 내 활동과 관련된 알림을 받을 수 있다.
- **알림 유형:**
  - 내 게시물에 좋아요 → "{사용자}님이 회원님의 게시물을 좋아합니다."
  - 내 게시물에 댓글 → "{사용자}님이 댓글을 남겼습니다: {댓글 미리보기}"
  - 새로운 팔로워 → "{사용자}님이 회원님을 팔로우하기 시작했습니다."
  - 팔로우 요청 승인 → "{사용자}님이 회원님의 팔로우 요청을 수락했습니다."
  - 게시물/댓글에서 멘션 → "{사용자}님이 댓글에서 회원님을 언급했습니다."

### US-7.2: 알림 목록
- **사용자로서** 알림 목록을 확인할 수 있다.
- **인수 조건:**
  - 시간순 정렬
  - 읽음/안읽음 상태 표시
  - 알림 클릭 시 해당 게시물/프로필로 이동
  - 미읽은 알림 배지 (네비게이션)

### US-7.3: 실시간 알림
- **사용자로서** 앱을 사용 중일 때 실시간으로 알림을 받을 수 있다.
- **인수 조건:**
  - WebSocket 기반 실시간 푸시
  - 토스트 또는 드롭다운으로 새 알림 표시
  - 알림 배지 실시간 업데이트

---

## 8. 사용자 프로필 페이지

### US-8.1: 프로필 조회
- **사용자로서** 내/타인의 프로필 페이지를 볼 수 있다.
- **인수 조건:**
  - 프로필 사진, 사용자명, 이름, 자기소개
  - 게시물 수, 팔로워 수, 팔로잉 수
  - 게시물 그리드 (3열)
  - 본인 프로필: "프로필 편집" 버튼
  - 타인 프로필: "팔로우/언팔로우" 버튼
  - 비공개 계정 미팔로우 시 게시물 비표시

## 비기능 요구사항 (NFR)

### NFR-1: 성능
- 페이지 초기 로딩 시간 3초 이내 (LCP)
- 이미지 Lazy Loading 적용
- 피드 API 응답 시간 200ms 이내 (p95)

### NFR-2: 보안
- HTTPS 필수
- XSS, CSRF 방지
- 이미지 업로드 시 파일 타입 검증 (매직 바이트)
- Rate Limiting (API별)
- SQL Injection 방지 (ORM/Parameterized Query)

### NFR-3: 확장성
- 이미지 스토리지 분리 (Object Storage)
- DB 읽기 부하 분산 가능 구조
- Stateless 서버 (수평 확장 가능)

### NFR-4: 접근성
- 반응형 웹 (모바일, 태블릿, 데스크톱)
- 이미지 alt 텍스트 지원
- 키보드 내비게이션 지원
- WCAG 2.1 AA 준수

### NFR-5: 데이터
- 소프트 삭제 적용 (게시물, 댓글, 계정)
- 사용자 데이터 내보내기 기능 (GDPR 대응)
- 일일 DB 백업
