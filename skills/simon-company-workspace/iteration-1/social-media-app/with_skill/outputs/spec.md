# Spec: InstaClone - 소셜 미디어 웹앱

**Status**: Reviewed
**Created**: 2026-03-07

## 1. User Stories & Scenarios

### US-1: 이메일 회원가입 및 로그인 (P1)

사용자가 서비스를 이용하려면 계정이 필요하다. 이메일과 비밀번호로 회원가입하고 로그인할 수 있어야 한다.

**Why this priority**: 모든 기능의 전제 조건. 계정 없이는 어떤 기능도 사용 불가.
**Independent Test**: 가입 → 로그인 → 로그아웃 흐름만으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 미가입 사용자, **When** 이메일/비밀번호/닉네임을 입력하고 가입 버튼을 누르면, **Then** 계정이 생성되고 로그인 상태가 된다.
2. **Given** 가입된 사용자, **When** 올바른 이메일/비밀번호를 입력하고 로그인하면, **Then** 인증되어 피드 화면으로 이동한다.
3. **Given** 로그인 상태의 사용자, **When** 로그아웃을 누르면, **Then** 세션이 종료되고 로그인 화면으로 이동한다.
4. **Given** 미가입 이메일로 로그인 시도, **When** 로그인 버튼을 누르면, **Then** "이메일 또는 비밀번호가 올바르지 않습니다" 에러 메시지가 표시된다.
5. **Given** 이미 가입된 이메일로 회원가입 시도, **When** 가입 버튼을 누르면, **Then** "이미 사용 중인 이메일입니다" 에러 메시지가 표시된다.

---

### US-2: 사진 게시물 업로드 (P1)

사용자가 사진을 선택하고 캡션을 작성하여 게시물을 올릴 수 있어야 한다.

**Why this priority**: 소셜 미디어의 핵심 기능. 콘텐츠가 없으면 피드도 의미가 없다.
**Independent Test**: 로그인 후 사진 업로드 → 게시물 생성 확인으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 로그인한 사용자, **When** 사진(JPEG/PNG)을 선택하고 캡션을 입력한 뒤 게시 버튼을 누르면, **Then** 게시물이 생성되고 내 프로필에 표시된다.
2. **Given** 로그인한 사용자, **When** 사진 없이 게시를 시도하면, **Then** "사진을 선택해주세요" 에러가 표시된다.
3. **Given** 로그인한 사용자, **When** 10MB 초과 파일을 업로드하면, **Then** "파일 크기는 10MB 이하여야 합니다" 에러가 표시된다.
4. **Given** 로그인한 사용자, **When** 캡션 없이 사진만 업로드하면, **Then** 캡션 없이 게시물이 정상 생성된다.

---

### US-3: 피드 조회 (P1)

사용자가 팔로우한 사람들의 게시물을 최신순으로 볼 수 있어야 한다.

**Why this priority**: 소셜 미디어의 핵심 경험. 콘텐츠 소비의 메인 채널.
**Independent Test**: 팔로우한 사용자의 게시물이 최신순으로 피드에 나타나는지 확인.

**Acceptance Scenarios**:
1. **Given** 팔로우한 사용자가 있는 로그인 사용자, **When** 피드 화면에 진입하면, **Then** 팔로우한 사용자들의 게시물이 최신순으로 표시된다.
2. **Given** 아무도 팔로우하지 않은 사용자, **When** 피드 화면에 진입하면, **Then** "팔로우한 사용자가 없습니다. 사용자를 찾아보세요!" 안내 메시지가 표시된다.
3. **Given** 피드에 게시물이 많은 경우, **When** 스크롤을 아래로 내리면, **Then** 다음 게시물이 무한 스크롤로 로드된다.
4. **Given** 피드에 새 게시물이 있는 경우, **When** 피드를 새로고침(pull-to-refresh)하면, **Then** 최신 게시물이 상단에 추가된다.

---

### US-4: 사용자 팔로우/언팔로우 (P1)

다른 사용자를 팔로우하여 그 사용자의 게시물을 피드에서 볼 수 있어야 한다.

**Why this priority**: 팔로우 관계가 없으면 피드에 콘텐츠가 없어 서비스 가치가 없다.
**Independent Test**: 팔로우 → 팔로우 확인 → 언팔로우 → 언팔로우 확인 흐름으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 로그인한 사용자, **When** 다른 사용자의 프로필에서 팔로우 버튼을 누르면, **Then** 팔로우 관계가 생성되고 버튼이 "언팔로우"로 바뀐다.
2. **Given** 팔로우 중인 사용자, **When** 언팔로우 버튼을 누르면, **Then** 팔로우 관계가 해제되고 버튼이 "팔로우"로 바뀐다.
3. **Given** 자기 자신의 프로필, **When** 프로필을 조회하면, **Then** 팔로우/언팔로우 버튼이 표시되지 않는다.

---

### US-5: 좋아요 (P1)

게시물에 좋아요를 눌러 관심을 표현할 수 있어야 한다.

**Why this priority**: 소셜 미디어의 핵심 인터랙션. 콘텐츠에 대한 피드백의 가장 기본적인 형태.
**Independent Test**: 좋아요 → 좋아요 수 증가 → 좋아요 취소 → 좋아요 수 감소로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 로그인한 사용자와 게시물, **When** 좋아요 버튼을 누르면, **Then** 좋아요가 등록되고 좋아요 수가 1 증가하며 버튼이 활성(채워진 하트) 상태로 바뀐다.
2. **Given** 이미 좋아요한 게시물, **When** 좋아요 버튼을 다시 누르면, **Then** 좋아요가 취소되고 좋아요 수가 1 감소하며 버튼이 비활성(빈 하트) 상태로 바뀐다.
3. **Given** 좋아요가 N개인 게시물, **When** 게시물을 조회하면, **Then** 정확한 좋아요 수(N)가 표시된다.

---

### US-6: 댓글 (P1)

게시물에 텍스트 댓글을 작성하고 조회할 수 있어야 한다.

**Why this priority**: 소셜 미디어의 핵심 인터랙션. 콘텐츠에 대한 대화와 소통.
**Independent Test**: 댓글 작성 → 댓글 목록 조회 → 댓글 삭제 흐름으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 로그인한 사용자와 게시물, **When** 댓글 입력 필드에 텍스트를 입력하고 제출하면, **Then** 댓글이 게시물 하단에 추가되고 댓글 수가 1 증가한다.
2. **Given** 댓글이 있는 게시물, **When** 게시물의 댓글 영역을 조회하면, **Then** 모든 댓글이 최신순으로 표시되며 작성자 닉네임과 작성 시간이 함께 보인다.
3. **Given** 자신이 작성한 댓글, **When** 삭제 버튼을 누르면, **Then** 댓글이 삭제되고 댓글 수가 1 감소한다.
4. **Given** 빈 텍스트로 댓글 제출 시도, **When** 제출 버튼을 누르면, **Then** "댓글 내용을 입력해주세요" 에러가 표시된다.

---

### US-7: 사용자 프로필 (P1)

사용자의 프로필 페이지에서 프로필 정보와 게시물 목록을 볼 수 있어야 한다.

**Why this priority**: 사용자 정체성 표현과 게시물 관리의 기본. 팔로우 판단의 근거.
**Independent Test**: 내 프로필 조회 → 게시물 목록 확인 → 팔로워/팔로잉 수 확인으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 로그인한 사용자, **When** 프로필 페이지에 진입하면, **Then** 프로필 사진, 닉네임, 자기소개, 게시물 수, 팔로워 수, 팔로잉 수가 표시된다.
2. **Given** 프로필 페이지, **When** 게시물 영역을 보면, **Then** 해당 사용자의 게시물이 그리드 형태로 표시된다.
3. **Given** 다른 사용자의 프로필, **When** 팔로워/팔로잉 수를 클릭하면, **Then** 팔로워/팔로잉 목록이 표시된다.

---

### US-8: 알림 (P1)

다른 사용자가 나에게 좋아요, 댓글, 팔로우를 했을 때 알림을 받을 수 있어야 한다.

**Why this priority**: 사용자 참여도와 리텐션의 핵심. 인터랙션에 대한 피드백 루프.
**Independent Test**: 특정 이벤트 발생 → 알림 목록에서 확인으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 내 게시물에 다른 사용자가 좋아요를 누르면, **When** 알림 목록을 확인하면, **Then** "[닉네임]님이 회원님의 게시물을 좋아합니다" 알림이 표시된다.
2. **Given** 내 게시물에 다른 사용자가 댓글을 달면, **When** 알림 목록을 확인하면, **Then** "[닉네임]님이 댓글을 남겼습니다: [댓글 미리보기]" 알림이 표시된다.
3. **Given** 다른 사용자가 나를 팔로우하면, **When** 알림 목록을 확인하면, **Then** "[닉네임]님이 회원님을 팔로우하기 시작했습니다" 알림이 표시된다.
4. **Given** 읽지 않은 알림이 있는 경우, **When** 헤더의 알림 아이콘을 보면, **Then** 읽지 않은 알림 개수가 배지로 표시된다.
5. **Given** 알림 목록에서 알림을 클릭하면, **When** 해당 알림의 대상(게시물 또는 프로필)으로 이동하면, **Then** 해당 알림이 읽음 처리된다.

---

### US-9: 프로필 편집 (P2)

사용자가 자신의 프로필 사진, 닉네임, 자기소개를 수정할 수 있어야 한다.

**Why this priority**: 기본 MVP 이후 자기 표현의 확장. P1 없이는 의미가 없으므로 P2.
**Independent Test**: 프로필 수정 → 변경 사항 확인으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 로그인한 사용자, **When** 프로필 편집 페이지에서 닉네임을 변경하고 저장하면, **Then** 프로필에 변경된 닉네임이 표시된다.
2. **Given** 로그인한 사용자, **When** 프로필 사진을 변경하면, **Then** 새 프로필 사진이 업로드되어 표시된다.
3. **Given** 이미 사용 중인 닉네임으로 변경 시도, **When** 저장하면, **Then** "이미 사용 중인 닉네임입니다" 에러가 표시된다.

---

### US-10: 사용자 검색 (P2)

닉네임으로 다른 사용자를 검색하여 프로필을 찾을 수 있어야 한다.

**Why this priority**: 팔로우할 사용자를 발견하는 수단. P1 팔로우 기능을 보완.
**Independent Test**: 검색어 입력 → 결과 목록 표시 → 프로필 이동으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 검색 화면, **When** 닉네임을 입력하면, **Then** 해당 닉네임을 포함하는 사용자 목록이 표시된다.
2. **Given** 검색 결과 목록, **When** 사용자를 클릭하면, **Then** 해당 사용자의 프로필 페이지로 이동한다.
3. **Given** 존재하지 않는 닉네임 검색, **When** 검색하면, **Then** "검색 결과가 없습니다" 메시지가 표시된다.

---

### US-11: 게시물 삭제 (P2)

자신이 올린 게시물을 삭제할 수 있어야 한다.

**Why this priority**: 콘텐츠 관리 기능. MVP 이후 필요하지만 핵심 흐름은 아님.
**Independent Test**: 게시물 삭제 → 프로필에서 사라짐 확인으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 자신의 게시물, **When** 삭제 버튼을 누르고 확인하면, **Then** 게시물이 삭제되고 프로필 게시물 목록에서 사라진다.
2. **Given** 다른 사용자의 게시물, **When** 게시물을 조회하면, **Then** 삭제 버튼이 표시되지 않는다.

---

### US-12: 게시물 상세 보기 (P2)

개별 게시물의 상세 페이지에서 큰 사진, 캡션, 좋아요, 댓글을 모두 볼 수 있어야 한다.

**Why this priority**: 피드에서는 축약된 정보만 보이므로 상세 보기 필요. P1 기능의 보완.
**Independent Test**: 게시물 클릭 → 상세 페이지에서 모든 정보 확인으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 피드의 게시물, **When** 게시물을 클릭하면, **Then** 상세 페이지에서 큰 사진, 캡션, 좋아요 수, 전체 댓글, 작성 시간이 표시된다.
2. **Given** 상세 페이지, **When** 좋아요 버튼이나 댓글을 입력하면, **Then** 피드에서와 동일하게 동작한다.

---

### US-13: 탐색 페이지 (P3)

팔로우하지 않은 사용자의 인기 게시물을 탐색할 수 있어야 한다.

**Why this priority**: 콘텐츠 발견과 네트워크 확장. MVP 이후 성장에 기여.
**Independent Test**: 탐색 페이지 진입 → 인기 게시물 그리드 표시로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 로그인한 사용자, **When** 탐색 페이지에 진입하면, **Then** 최근 인기 게시물(좋아요 수 기준)이 그리드로 표시된다.
2. **Given** 탐색 페이지의 게시물, **When** 클릭하면, **Then** 해당 게시물의 상세 페이지로 이동한다.

---

### US-14: 비밀번호 변경 (P3)

사용자가 비밀번호를 변경할 수 있어야 한다.

**Why this priority**: 보안 기능이지만 MVP 이후 구현해도 무방.
**Independent Test**: 현재 비밀번호 확인 → 새 비밀번호 설정 → 새 비밀번호로 로그인으로 독립 검증 가능.

**Acceptance Scenarios**:
1. **Given** 로그인한 사용자, **When** 현재 비밀번호를 올바르게 입력하고 새 비밀번호를 설정하면, **Then** 비밀번호가 변경되고 성공 메시지가 표시된다.
2. **Given** 현재 비밀번호를 틀리게 입력한 경우, **When** 변경을 시도하면, **Then** "현재 비밀번호가 올바르지 않습니다" 에러가 표시된다.

---

## 2. Functional Requirements

- **FR-001**: 시스템은 이메일/비밀번호 기반 회원가입 및 로그인을 지원해야 한다
- **FR-002**: 사용자는 JPEG/PNG 형식의 사진(최대 10MB)을 업로드하여 게시물을 생성할 수 있어야 한다
- **FR-003**: 시스템은 팔로우한 사용자의 게시물을 최신순으로 정렬하여 피드에 표시해야 한다
- **FR-004**: 사용자는 다른 사용자를 팔로우/언팔로우할 수 있어야 한다
- **FR-005**: 사용자는 게시물에 좋아요를 토글(등록/취소)할 수 있어야 한다
- **FR-006**: 사용자는 게시물에 텍스트 댓글을 작성하고, 자신의 댓글을 삭제할 수 있어야 한다
- **FR-007**: 시스템은 좋아요, 댓글, 팔로우 이벤트 발생 시 대상 사용자에게 인앱 알림을 생성해야 한다
- **FR-008**: 사용자 프로필 페이지에서 프로필 정보(사진, 닉네임, 자기소개)와 게시물 그리드를 표시해야 한다
- **FR-009**: 시스템은 닉네임 기반 사용자 검색을 지원해야 한다
- **FR-010**: 시스템은 무한 스크롤을 통한 피드 페이지네이션을 지원해야 한다
- **FR-011**: 시스템은 업로드된 이미지를 적절한 크기로 리사이즈하여 저장해야 한다 (원본 + 썸네일)
- **FR-012**: 비밀번호는 해시 처리하여 저장해야 한다
- **FR-013**: 인증된 사용자만 게시물 생성, 좋아요, 댓글, 팔로우 기능을 이용할 수 있어야 한다

## 3. Edge Cases

- **EC-001**: 동시에 같은 게시물에 좋아요를 여러 번 빠르게 누른 경우 — 중복 좋아요가 생기지 않아야 한다 (idempotent)
- **EC-002**: 이미 삭제된 게시물에 좋아요/댓글을 시도하는 경우 — 적절한 에러 메시지 표시
- **EC-003**: 자기 자신을 팔로우하려는 경우 — 차단하고 팔로우 버튼을 표시하지 않음
- **EC-004**: 네트워크 오류로 이미지 업로드가 중단된 경우 — 부분 업로드 데이터 정리 및 재시도 안내
- **EC-005**: 매우 긴 댓글(1000자 이상) 입력 시도 — 글자 수 제한 표시 및 차단
- **EC-006**: 지원하지 않는 이미지 형식(GIF, BMP, WebP 등) 업로드 시도 — "JPEG 또는 PNG 파일만 업로드할 수 있습니다" 에러
- **EC-007**: 팔로우한 사용자가 계정을 삭제한 경우 — 해당 사용자의 게시물이 피드에서 자연스럽게 제거

## 4. Key Entities

- **User**: 서비스 사용자. 이메일, 닉네임, 프로필 사진, 자기소개를 가진다. 다른 User를 팔로우할 수 있다.
- **Post**: 사용자가 올린 게시물. 반드시 하나의 사진을 포함하며, 선택적으로 캡션을 가진다. 하나의 User가 소유한다.
- **Like**: User와 Post 간의 좋아요 관계. 한 User는 한 Post에 한 번만 좋아요 가능.
- **Comment**: User가 Post에 남긴 텍스트 댓글. 작성자와 작성 시간을 가진다.
- **Follow**: User 간의 팔로우 관계. 단방향(A→B). A가 B를 팔로우하면 B의 게시물이 A의 피드에 표시된다.
- **Notification**: 좋아요, 댓글, 팔로우 이벤트로 생성되는 알림. 수신자, 발신자, 이벤트 유형, 대상(Post/User), 읽음 여부를 가진다.

## 5. Success Criteria

- **SC-001**: 신규 사용자가 3분 이내에 회원가입 → 사진 업로드 → 게시물 생성을 완료할 수 있다
- **SC-002**: 피드 로딩 시간이 500ms 이내이다 (100개 이하 게시물 기준)
- **SC-003**: 좋아요/댓글 반응 시간이 200ms 이내이다
- **SC-004**: 동시 접속자 100명 기준 서비스가 정상 동작한다
- **SC-005**: 알림이 이벤트 발생 후 5초 이내에 생성된다
- **SC-006**: 업로드된 이미지의 썸네일이 1초 이내에 생성된다
