## Walking Skeleton (Release 1 - Sprint 1 목표)

Walking Skeleton은 각 Activity의 최소 필수 Task를 수평으로 연결한 것이다. 핵심 흐름이 끝에서 끝까지 동작하는 최소 버전:

**핵심 흐름**: 가입 → 사진 업로드 → 팔로우 → 피드에서 게시물 확인 → 좋아요/댓글 → 알림 확인

| Activity | 최소 필수 Story | 설명 |
|----------|----------------|------|
| 가입/인증 | US-1 | 이메일 가입/로그인으로 서비스 진입 |
| 콘텐츠 생성 | US-2 | 사진 업로드로 게시물 생성 |
| 콘텐츠 소비 | US-3 + US-7 | 피드에서 최신순 게시물 소비 + 프로필에서 게시물 확인 |
| 소셜 인터랙션 | US-4 + US-5 + US-6 | 팔로우로 관계 형성, 좋아요/댓글로 소통 |
| 알림/관리 | US-8 | 좋아요/댓글/팔로우 알림으로 피드백 루프 |

Walking Skeleton이 구현되면 "가입 → 사진 올리기 → 다른 사용자 팔로우 → 피드에서 사진 보기 → 좋아요/댓글 → 알림 확인"의 전체 사용자 여정이 동작한다.

## Release 계획

### Release 1 (Walking Skeleton) — P1 Stories

**목표**: 핵심 흐름이 끝에서 끝까지 동작하는 최소 버전

| Sprint | Stories | 팀 | 비고 |
|--------|---------|-----|------|
| Sprint 1 | 프로젝트 초기 설정 + DB 스키마 + US-1 (가입/로그인) | Backend, Frontend, DBA, DevOps | 기반 인프라 + 인증 |
| Sprint 2 | US-2 (사진 업로드) + US-7 (프로필) + US-4 (팔로우) | Backend, Frontend | 콘텐츠 생성 + 소셜 관계 |
| Sprint 3 | US-3 (피드) + US-5 (좋아요) + US-6 (댓글) + US-8 (알림) | Backend, Frontend | 콘텐츠 소비 + 인터랙션 + 알림 |

### Release 3 — P3 Stories

                        │

**설명**:
- US-1(가입/로그인)은 모든 스토리의 전제 조건
- US-2(사진 업로드)는 US-3(피드), US-5(좋아요), US-6(댓글)의 전제 (게시물이 있어야 함)
- US-4(팔로우)는 US-3(피드)의 전제 (팔로우해야 피드에 나타남)
- US-5(좋아요), US-6(댓글), US-4(팔로우)는 US-8(알림)의 전제 (이벤트 소스)
- US-7(프로필)은 US-1 이후 독립적으로 구현 가능 (사용자 정보 + 게시물 조회)
