# Constitution: ML Recommender (사용자 행동 기반 상품 추천 시스템)

**Created**: 2026-03-07
**Last Updated**: 2026-03-07 (Phase 1-D 완성)
**Status**: Approved

## Core Principles

1. **추천 품질 최우선**: 사용자에게 관련성 높은 상품을 추천하는 것이 시스템의 존재 이유다. 기술적 편의를 위해 추천 품질을 타협하지 않는다.
2. **서비스 무중단**: 모델 학습, 배포, 장애 상황에서도 추천 API는 항상 응답해야 한다. 폴백 메커니즘을 반드시 구비한다.
3. **데이터 기반 의사결정**: 추천 성능 개선은 반드시 측정 가능한 지표(CTR, 전환율, 커버리지)에 기반해야 한다. 직관이 아닌 데이터로 판단한다.
4. **보안은 타협 불가**: 관리자 대시보드 인증, API 인가, 사용자 데이터 보호는 어떤 경우에도 우회하지 않는다.
5. **재현 가능한 ML**: 모델 학습은 반드시 재현 가능해야 한다. 학습 데이터, 하이퍼파라미터, 랜덤 시드를 기록하고 모델 버전을 관리한다.

## Quality Gates

- 테스트 커버리지: 80% 이상
- TRP 3라운드 통과 필수
- 추천 API p95 응답 시간: 200ms 이내
- 이벤트 수집 API 처리량: 초당 1000건 이상
- 추천 커버리지: 70% 이상
- OWASP Top 10 준수
- 모델 학습 재현성: 동일 데이터 + 시드 → 동일 결과
- ML 모델 평가 지표 기준: Precision@10 >= 0.1 (최소 기준, 데이터 축적에 따라 상향)

## Constraints

### 기술적 제약
- Collaborative Filtering 기반 추천 알고리즘 사용 (사용자 명시 요구)
- REST API로 추천 결과 제공 (사용자 명시 요구)
- 관리자 대시보드는 웹 기반
- Cold Start 문제에 대한 폴백 메커니즘 필수

### 운영 제약
- 모델 재학습 중 기존 모델로 계속 서빙 (무중단)
- 모델 롤백이 즉시 가능해야 함
- 행동 이벤트 중복 제거 로직 필수

### 데이터 제약
- 사용자 행동 데이터의 최소 보존 기간: 90일
- 개인 식별 정보(PII) 최소화: 추천에 필요한 행동 데이터만 수집
- 사용자 ID는 외부 시스템의 식별자를 그대로 사용 (자체 회원가입 없음)

## DoR (Definition of Ready) - 작업 시작 전 필수 조건

- [ ] INVEST 기준 충족 (Story가 독립적이고, 추정 가능하고, 테스트 가능)
- [ ] Acceptance Criteria 명시 (Given/When/Then)
- [ ] 관련 Contract 정의 완료
- [ ] 의존 Feature 완료 또는 Mock 준비
- [ ] 파일 소유권 확정

## DoD (Definition of Done) - 작업 완료 기준

- [ ] 코드 구현 + 테스트 통과 (커버리지 80%+)
- [ ] Contract 준수 확인
- [ ] TRP R1-R3 통과
- [ ] 빌드 성공 + 린트 통과
- [ ] 기술 부채 없음 (또는 명시적 기록)
- [ ] ML 모델의 경우: 평가 지표가 기준치 이상
