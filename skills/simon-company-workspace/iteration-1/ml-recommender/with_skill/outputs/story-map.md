## Walking Skeleton (Release 1 - Sprint 1 목표)

각 Activity의 최소 필수 Task를 수평으로 연결한 핵심 흐름의 최소 구현:

```
[상품 등록] → [행동 이벤트 수집] → [CF 모델 학습] → [추천 API 호출] → [대시보드 로그인] → [성능 지표 확인]
  US-3           US-1                US-4              US-2              US-6               US-5
```

**Walking Skeleton이 검증하는 것:**
1. 상품 데이터가 시스템에 등록될 수 있는가?
2. 사용자 행동 데이터가 수집/저장되는가?
3. 수집된 데이터로 CF 모델이 학습되는가?
4. 학습된 모델로 추천 결과가 생성되는가?
5. 관리자가 인증 후 대시보드에서 지표를 볼 수 있는가?

이 흐름이 end-to-end로 동작하면, 이후 P2/P3 기능을 안전하게 추가할 수 있다.

### Walking Skeleton 의존성 그래프

```
     US-3 (상품 카탈로그)       US-6 (관리자 인증)
       |                            |
     US-1 (행동 수집)  ----+        |
       |                   |        |
     US-4 (모델 학습)      |        |
       |                   |        |
     US-2 (추천 API) ------+--------+
                           |
                     US-5 (대시보드)
```

**병렬 가능 그룹:**
- Group A (독립): US-3 + US-1 + US-6 -- 모두 동시에 개발 시작 가능
- Group B (US-1 의존): US-4 -- 행동 데이터 수집이 준비되어야 학습 가능
- Group C (US-4 의존): US-2 -- 학습된 모델이 있어야 서빙 가능
- Group D (US-2 + US-6 의존): US-5 -- 추천 데이터와 인증이 모두 필요

## Release 계획

### Release 1 (Walking Skeleton): P1 핵심 흐름
- US-1: 사용자 행동 데이터 수집 API
- US-2: 상품 추천 API (cold start 폴백 포함)
- US-3: 상품 카탈로그 관리 API
- US-4: Collaborative Filtering 모델 학습
- US-5: 추천 성능 지표 대시보드 (기본 4개 지표)
- US-6: 관리자 인증

**가치**: 추천 시스템의 전체 흐름이 동작하고, 관리자가 성능을 모니터링할 수 있다.

### Release 2: P2 운영 기능 강화
- US-7: 모델 학습 스케줄링 및 수동 트리거
- US-8: A/B 테스트 관리
- US-9: 추천 제외/부스트 규칙 관리

**가치**: 관리자가 모델을 능동적으로 관리하고, 비즈니스 규칙으로 추천을 제어할 수 있다.

### Release 3: P3 고도화
- US-10: 추천 결과 설명
- US-11: 실시간 이벤트 스트림 대시보드

**가치**: 추천 투명성과 운영 가시성이 극대화된다.
