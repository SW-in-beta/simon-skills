# 프로젝트 기획 산출물 요약

## 프로젝트

사용자 행동 기반 상품 추천 시스템 (Collaborative Filtering)

---

## 생성된 산출물

### 1. 팀 구성 및 역할 정의
- **파일**: `01-team-composition.md`
- **내용**: 7~9명 규모의 팀 구성안. PM, PO, ML 엔지니어 2명, 백엔드 엔지니어 2명, 프론트엔드 엔지니어 1명, 데이터 엔지니어 1명, QA 엔지니어 1명(선택)으로 구성. 각 역할별 필요 역량, 주요 산출물, 세부 역할 분담, 조직 구조도, 외부 협력 필요 영역 정의.

### 2. 기능 명세서
- **파일**: `02-feature-specifications.md`
- **내용**: 5개 영역(데이터 수집/처리, 추천 모델, REST API, 관리자 대시보드, 시스템 아키텍처)에 대한 상세 기능 명세. 데이터 스키마, ALS/NCF 모델 설명, 모델 평가 지표 및 목표값, API 엔드포인트 5종(개인화 추천, 유사 상품, 이벤트 수집, 배치 추천, 피드백)의 요청/응답 스펙, 비기능 요구사항(SLA, 응답시간, Rate Limit), 대시보드 5개 페이지 구성 및 알림 규칙, 전체 기술 스택 정리.

### 3. 사용자 스토리
- **파일**: `03-user-stories.md`
- **내용**: 5개 에픽, 21개 사용자 스토리. 각 스토리는 역할/원함/이유 형식으로 작성되며, 체크리스트 형태의 수락 기준과 스토리 포인트를 포함. MoSCoW 우선순위(Must Have 11개, Should Have 6개, Could Have 4개, Won't Have 2개)로 백로그 분류.

### 4. 프로젝트 계획서
- **파일**: `04-project-plan.md`
- **내용**: 16주(8 스프린트) 타임라인. 5개 Phase(셋업 → 데이터/모델 → API/대시보드 MVP → 고도화 → 릴리스)로 구분. 스프린트별 태스크(총 48개)와 담당자/의존성 배정, 팀원별 워크로드 요약, 7개 리스크 식별 및 완화 전략, MVP/GA 성공 기준, 커뮤니케이션 계획, 의존성 다이어그램, 기술 부채 관리 방침.
