# 팀 구성 및 역할 정의

## 프로젝트 개요

사용자 행동 기반 상품 추천 시스템 (Collaborative Filtering)을 구축하기 위한 팀 구성안.

---

## 필요 인원: 7~9명

### 1. 프로젝트 매니저 (PM) - 1명

| 항목 | 내용 |
|------|------|
| 역할 | 프로젝트 일정/리소스 관리, 이해관계자 커뮤니케이션, 리스크 관리 |
| 필요 역량 | 애자일 프로젝트 관리 경험, ML 프로젝트 라이프사이클 이해 |
| 주요 산출물 | 프로젝트 일정표, 주간 보고서, 리스크 레지스터 |

### 2. 프로덕트 오너 / 기획자 (PO) - 1명

| 항목 | 내용 |
|------|------|
| 역할 | 비즈니스 요구사항 정의, 우선순위 결정, 사용자 스토리 작성 |
| 필요 역량 | 추천 시스템 도메인 지식, 데이터 기반 의사결정 경험 |
| 주요 산출물 | PRD, 사용자 스토리, 수락 기준 |

### 3. ML 엔지니어 - 2명

| 항목 | 내용 |
|------|------|
| 역할 | 추천 모델 설계/개발/학습, 모델 성능 최적화, A/B 테스트 설계 |
| 필요 역량 | Collaborative Filtering (ALS, SVD, Neural CF) 구현 경험, Python, PyTorch/TensorFlow, Spark |
| 주요 산출물 | 추천 모델, 학습 파이프라인, 모델 성능 리포트 |

#### 세부 역할 분담
- **ML 엔지니어 (시니어)**: 모델 아키텍처 설계, 오프라인 평가 체계 구축, 모델 서빙 파이프라인
- **ML 엔지니어 (주니어/미드)**: 데이터 전처리, 피처 엔지니어링, 실험 수행 및 결과 분석

### 4. 백엔드 엔지니어 - 2명

| 항목 | 내용 |
|------|------|
| 역할 | REST API 설계/개발, 데이터 파이프라인 구축, 인프라 설정 |
| 필요 역량 | Python(FastAPI/Django) 또는 Go, Redis/PostgreSQL, 대용량 트래픽 처리 경험 |
| 주요 산출물 | API 서버, 데이터 수집 파이프라인, 캐싱 레이어 |

#### 세부 역할 분담
- **백엔드 엔지니어 A**: REST API 서버, 인증/인가, 캐싱 전략
- **백엔드 엔지니어 B**: 사용자 행동 데이터 수집 파이프라인, 배치 처리, 모델 서빙 인프라

### 5. 프론트엔드 엔지니어 - 1명

| 항목 | 내용 |
|------|------|
| 역할 | 관리자 대시보드 개발 |
| 필요 역량 | React/Next.js, 데이터 시각화 (Chart.js, Recharts 등), 반응형 UI |
| 주요 산출물 | 관리자 대시보드 웹 애플리케이션 |

### 6. 데이터 엔지니어 - 1명

| 항목 | 내용 |
|------|------|
| 역할 | 데이터 웨어하우스 설계, ETL 파이프라인 구축, 데이터 품질 관리 |
| 필요 역량 | Airflow, Spark, BigQuery/Redshift, 데이터 모델링 |
| 주요 산출물 | ETL 파이프라인, 데이터 스키마, 데이터 품질 모니터링 |

### 7. QA 엔지니어 - 1명 (선택)

| 항목 | 내용 |
|------|------|
| 역할 | 테스트 전략 수립, API/대시보드 테스트, 성능 테스트 |
| 필요 역량 | API 테스트 자동화, 부하 테스트 (k6, Locust), ML 모델 검증 |
| 주요 산출물 | 테스트 계획서, 테스트 자동화 스크립트, 성능 테스트 리포트 |

---

## 조직 구조

```
PM
├── PO (기획)
├── ML팀
│   ├── ML 엔지니어 (시니어)
│   └── ML 엔지니어 (주니어/미드)
├── 개발팀
│   ├── 백엔드 엔지니어 A (API)
│   ├── 백엔드 엔지니어 B (데이터/인프라)
│   └── 프론트엔드 엔지니어 (대시보드)
├── 데이터 엔지니어
└── QA 엔지니어 (선택)
```

## 외부 협력 필요 영역

| 영역 | 협력 대상 | 내용 |
|------|-----------|------|
| 인프라/DevOps | 플랫폼팀 | K8s 클러스터, CI/CD 파이프라인, GPU 리소스 |
| 보안 | 보안팀 | 개인정보 처리 검토, API 보안 감사 |
| 디자인 | 디자인팀 | 관리자 대시보드 UI/UX 디자인 |
| 데이터 | 데이터팀 | 기존 사용자 행동 데이터 접근 권한 및 스키마 공유 |
